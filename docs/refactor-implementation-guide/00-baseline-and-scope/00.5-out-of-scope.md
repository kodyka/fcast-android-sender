# 00.5 — Out of scope

Things this refactor explicitly does **not** touch. Each item is deferred to a
future plan with the reason captured here.

## Slint UI tree (`ui/`)

The existing `ui/` directory — `bridge.slint`, all `ui/pages/*.slint`, all
`ui/state/*.slint` — is treated as fixed. The refactor only:

- Wires the new `RuntimeBridge` (step 05) into the existing `Bridge` globals.
- Maps the new Kotlin `UiState` (step 08) onto the existing `AppState` enum.

It does **not**:

- Restructure `bridge.slint` or split it further.
- Introduce new panels.
- Change theme tokens, colours, or font sizes.
- Touch the existing pre-commit hook set for Slint (`forbid-raw-hex-colors`,
  `forbid-hard-coded-font-size`, `forbid-direct-Bridge.active-panel-writes`,
  `forbid-direct-Bridge.lifecycle-writes`).

Reason: the report is a refactor of the Android/native boundary. The Slint UI
is healthy enough that touching it would add risk without addressing the
report's findings.

## Workspace crates

```text
crates/
├── migration-runtime/      ← unchanged
├── gstpop-runtime/         ← unchanged
└── …
vendor/
└── gstpop/                 ← unchanged
```

Step 07's sub-PR 7.6 **may** create new crates (`crates/runtime-core`,
`crates/android-jni`) that depend on the existing workspace members — but it
does not modify the internals of `crates/migration-runtime` or
`crates/gstpop-runtime`.

## External git dependencies

`Cargo.toml` pins to specific commits in `kodyka/fcast`. The refactor does
**not** bump or replace any of these pins. If a step requires a feature only
available in a newer pin, that bump is its own follow-up PR.

## GStreamer Android SDK

Pinned at 1.28.0. Step 11.7 (NDK r28c) explicitly does not attempt a
GStreamer version bump.

## Release signing material

Step 09 introduces the signing **path**, but the actual keystore lives outside
the repository. Provisioning the keystore is a human operation, not a guide
step.

## `MainActivity`'s NativeActivity contract

The class extends a `NativeActivity` so that `android_main` can run as the
Android entry point. The refactor keeps that contract; it does not migrate
the app to a pure Java/Kotlin `Activity`.
