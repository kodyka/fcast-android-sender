## C5 — Backend services call `nativeStart` on the service main thread

`GstPopService.java:36-56`:

```java
@Override
public int onStartCommand(Intent intent, int flags, int startId) {
    String action = intent != null ? intent.getAction() : null;
    if (ACTION_START.equals(action)) {
        startForeground(NOTIFICATION_ID, buildNotification("Starting gst-pop…"));
        String config = intent.getStringExtra(EXTRA_CONFIG_JSON);
        String statusJson = GstPopServiceBridge.nativeStart(config != null ? config : "");   // ← main thread
        // …
        return START_STICKY;
    }
    // …
}
```

`MigrationRuntimeService.java:36-56` is structurally identical. Verification:

```bash
rg -n 'nativeStart' app/src/main/java/org/fcast/android/sender/{GstPopService,MigrationRuntimeService}.java
```

Refactor: **Step 05** introduces a typed `RuntimeBridge` that wraps the call in
`Dispatchers.IO`. Step 05 does **not** change the service-side call site (kept
for future migration).
