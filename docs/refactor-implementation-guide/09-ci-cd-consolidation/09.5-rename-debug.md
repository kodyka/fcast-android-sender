# 09.5 — Rename `android-release-apk.yml` → `android-debug-apk.yml`

The file in `.github/workflows/android-release-apk.yml` builds a *debug*
APK despite its name. The new release workflow (sub-step 09.6) needs the
`android-release-apk.yml` filename, so rename the existing one first.

## Rename

```bash
git mv .github/workflows/android-release-apk.yml \
       .github/workflows/android-debug-apk.yml
```

## Update workflow name inside the file

```diff
- name: Android APK Release
+ name: Android Debug APK
```

If the workflow uses `if: github.event_name == 'release'` anywhere, this
is no longer accurate — it was never a release workflow despite the name.
Strip those checks; the file is now triggered only on `push` and
`pull_request`.

## Update badges and docs

Search for the old workflow name:

```bash
rg -n 'android-release-apk\.yml' .
rg -n 'Android APK Release\|Android Release APK' .
```

Likely hits:

- `README.md` — Status badge URL points to the renamed workflow.
- `docs/*.md` — Mentions of the workflow name.
- `.github/CODEOWNERS` (if any) — Path-based rules using the old filename.

Update each. Badge URL stays the same syntax, only the workflow filename
changes:

```markdown
- [![Android Debug APK](https://github.com/kodyka/fcast-android-sender/actions/workflows/android-debug-apk.yml/badge.svg)](https://github.com/kodyka/fcast-android-sender/actions/workflows/android-debug-apk.yml)
```

## Branch-protection considerations

If branch protection requires a status check named *"Android APK Release"*,
update it:

1. GitHub repo → Settings → Branches → main → Edit rule.
2. Status checks → remove `Android APK Release`.
3. Add `Android Debug APK` (or the new actual job name).

Otherwise the next PR will block on a check that no longer exists.

## Confirm rename

```bash
gh workflow list
```

Should list `Android Debug APK` (and no `Android APK Release`).

## Why "Low risk"

Renaming a workflow file is mechanical. The trigger (`push`, `pull_request`)
stays the same, the build steps stay the same. Only the filename and
display name change. The only failure mode is forgetting to update
branch-protection settings, which is fixable in 30 seconds.
