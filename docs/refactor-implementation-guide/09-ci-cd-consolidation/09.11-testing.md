## Detailed: cache-hit verification

After 9.A lands:

1. Push a PR. Watch the workflow run.
2. First run: every `actions/cache@v4` step reports `Cache miss` and the
   download steps execute.
3. Re-run the workflow without changing anything. Every cache step
   should report `Cache hit (key=…)` and the conditional download steps
   should be skipped.

If a cache step reports `Cache miss` on the second run with no change in
between, the cache key is unstable. Most common cause: a build artifact
inside the cached path with an ever-changing timestamp. Fix by excluding
the path with `path: |\n  …\n  !subdir`.

## Detailed: release dry-run

After 9.D lands:

```
Actions → Android Release APK → Run workflow
    Branch: main
    Version name: 0.0.0-dry-run
    Version code: 9999
    [Run workflow]
```

Wait for completion (~5–6 minutes with cache hits). Download the artifact.

```bash
unzip -p app-release-0.0.0-dry-run.zip app-release.apk > /tmp/app.apk
$ANDROID_HOME/build-tools/34.0.0/apksigner verify --verbose /tmp/app.apk
$ANDROID_HOME/build-tools/34.0.0/aapt dump badging /tmp/app.apk | grep version
```

Expected output:

```
Verified using v2 scheme (APK Signature Scheme v2): true
Verified using v3 scheme (APK Signature Scheme v3): true
versionCode='9999' versionName='0.0.0-dry-run'
```

If `versionName` is unset, the `-PversionName=$FCAST_VERSION_NAME` flag
didn't reach Gradle. Verify the `defaultConfig { versionName(…) }` block
in sub-step 09.7.

## Detailed: symbol-stability gate test

Land 9.E. Then deliberately break the gate to verify it works:

```bash
git checkout -b devin/test-symbol-stability
# Edit src/lib.rs:
#   pub extern "C" fn Java_..._nativeBackPressed → nativeBackPressed2
git commit -am "test: break symbol stability"
git push -u origin devin/test-symbol-stability
gh pr create --title "test: break symbol stability" --body "Should fail CI"
```

Wait for the workflow to run. The "Diff symbols" step should fail with:

```
< Java_org_fcast_android_sender_MainActivity_nativeBackPressed
> Java_org_fcast_android_sender_MainActivity_nativeBackPressed2
::error::JNI symbol set differs from base. …
```

Close the PR (don't merge).

## CI duration budget

After all of step 09 lands, target:

| Workflow                  | Pre-cache       | Post-cache       |
|---------------------------|------------------|------------------|
| `android-debug-apk.yml`   | ~9 min           | ~3 min           |
| `android-release-apk.yml` | ~10 min          | ~3.5 min         |
| `symbol-stability.yml`    | (2× build)       | ~5 min           |
| `gstpop-smoke.yml`        | ~2 min           | ~30 s            |
| `slint-viewer-smoke.yml`  | ~30 s            | ~5 s             |
| `ui-lint.yml`             | ~10 s            | ~5 s             |

If any workflow blows past its post-cache target by more than 2×, the
cache is missing or the build is doing redundant work. Inspect the
workflow log for "Cache miss" rows.
