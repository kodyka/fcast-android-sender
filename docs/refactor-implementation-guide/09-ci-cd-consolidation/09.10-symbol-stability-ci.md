# 09.10 — Symbol-stability CI gate

Per-PR guard for step 07's mandate that every `Java_org_fcast_android_sender_*`
symbol stays on the ABI surface.

Add a new workflow `.github/workflows/symbol-stability.yml`:

```yaml
name: JNI symbol stability

on:
  pull_request:
    paths:
      - 'src/**'
      - 'crates/**'
      - 'app/src/**'
      - 'ci/**'
      - '.github/workflows/symbol-stability.yml'

permissions:
  contents: read

jobs:
  diff-symbols:
    runs-on: ubuntu-24.04
    steps:
      - name: Check out PR
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Set up Android CI toolchain
        uses: ./.github/actions/android-ci-setup
        with:
          rust-target: aarch64-linux-android

      - name: Build .so from PR
        run: bash ci/build-rust-android-lib.sh debug

      - name: Dump PR symbols
        run: |
          mkdir -p /tmp/symbols
          nm -D --defined-only \
            target/aarch64-linux-android/debug/libfcastsender.so \
            | grep -E '^[0-9a-f]+ +T Java_org_fcast_android_sender_' \
            | awk '{print $3}' \
            | sort > /tmp/symbols/pr.txt
          cat /tmp/symbols/pr.txt

      - name: Check out base
        run: |
          git checkout ${{ github.event.pull_request.base.sha }}

      - name: Build .so from base
        run: bash ci/build-rust-android-lib.sh debug

      - name: Dump base symbols
        run: |
          nm -D --defined-only \
            target/aarch64-linux-android/debug/libfcastsender.so \
            | grep -E '^[0-9a-f]+ +T Java_org_fcast_android_sender_' \
            | awk '{print $3}' \
            | sort > /tmp/symbols/base.txt
          cat /tmp/symbols/base.txt

      - name: Diff symbols
        run: |
          if diff -u /tmp/symbols/base.txt /tmp/symbols/pr.txt; then
            echo "Symbols unchanged."
          else
            echo "::error::JNI symbol set differs from base. This breaks ABI compatibility for the Android side. See refactor step 07."
            exit 1
          fi
```

## Why a separate workflow?

- The debug pipeline already builds the .so but doesn't diff against
  base.
- Keeping the guard in its own file makes it skip-able for emergency
  hotfixes (admin override: re-run with `Allow merge without checks`).
- Its `paths:` filter ensures it doesn't run on doc-only PRs.

## What the guard catches

- A renamed JNI symbol (e.g., camelCase typo).
- A dropped JNI symbol (no `pub extern "C" fn …`).
- A new JNI symbol (the *new* symbol is also a diff; reviewers see it).

A *new* symbol failing the diff is intentional — the PR should be
explicit that it adds new JNI surface, and adding ABI surface is a
deliberate decision.

To allow a PR to add a symbol intentionally:

1. Open a follow-up commit on the same PR that adds the symbol to a
   tracked allowlist:
   ```
   # ci/jni-symbol-allowlist.txt
   Java_org_fcast_android_sender_MainActivity_nativeNewThing
   ```
2. Update the workflow to ignore allowed *new* symbols:
   ```bash
   diff -u <(sort /tmp/symbols/base.txt ci/jni-symbol-allowlist.txt) \
            /tmp/symbols/pr.txt
   ```

Until the allowlist exists, every new symbol fails the guard — which is
the right default during steps 07–08 when no new symbols are expected.

## Caching

The build runs twice (PR head and base). Caching makes the base build
fast on the second run because `Cargo.lock` is identical between
checkouts.

Use sub-step 09.4's caching block. The PR-vs-base difference adds at most
1–2 minutes to CI time.

## False positives

- Symbol order: sorted alphabetically, so commit-order is irrelevant.
- Function-body changes: don't change the symbol; not caught (which is
  fine — we only care about the symbol surface).
- Linker section differences: filtered by the `T` (text symbol) grep.

## Integration with branch protection

Add `JNI symbol stability / diff-symbols` to required status checks
(Settings → Branches → main → Edit rule).

After step 07's PR 7.2 lands, this check should pass on every subsequent
PR. If it fails, either:

- The PR genuinely changes the JNI surface (rare, intentional) — add an
  allowlist entry as described above.
- The PR accidentally renamed/dropped a symbol — fix the code.
