# 03.5 — `releaseEglResources()` helper

Extract the EGL/GL teardown so it can be called from `onDestroy` (and from
the capture-stop path) without duplication.

## New method

```java
private void releaseEglResources() {
    if (glHandler == null) {
        // Already torn down; nothing to do.
        return;
    }

    // Synchronously dispatch to the GL thread so glDelete* runs in the right
    // EGL context. If glThread.quitSafely was already called, the runnable
    // will simply not run — that is fine because the thread's exit also frees
    // the EGL context.
    final java.util.concurrent.CountDownLatch latch = new java.util.concurrent.CountDownLatch(1);
    boolean posted = glHandler.post(() -> {
        try {
            if (yFbo[0] != 0 || uFbo[0] != 0 || vFbo[0] != 0) {
                int[] fbos = new int[]{ yFbo[0], uFbo[0], vFbo[0] };
                android.opengl.GLES20.glDeleteFramebuffers(3, fbos, 0);
                yFbo[0] = uFbo[0] = vFbo[0] = 0;
            }
            if (yTex[0] != 0 || uTex[0] != 0 || vTex[0] != 0) {
                int[] texs = new int[]{ yTex[0], uTex[0], vTex[0] };
                android.opengl.GLES20.glDeleteTextures(3, texs, 0);
                yTex[0] = uTex[0] = vTex[0] = 0;
            }
            if (eglSurface != null && eglSurface != android.opengl.EGL14.EGL_NO_SURFACE) {
                android.opengl.EGL14.eglDestroySurface(eglDisplay, eglSurface);
                eglSurface = android.opengl.EGL14.EGL_NO_SURFACE;
            }
            if (eglContext != null && eglContext != android.opengl.EGL14.EGL_NO_CONTEXT) {
                android.opengl.EGL14.eglDestroyContext(eglDisplay, eglContext);
                eglContext = android.opengl.EGL14.EGL_NO_CONTEXT;
            }
            if (eglDisplay != null && eglDisplay != android.opengl.EGL14.EGL_NO_DISPLAY) {
                android.opengl.EGL14.eglTerminate(eglDisplay);
                eglDisplay = android.opengl.EGL14.EGL_NO_DISPLAY;
            }
        } catch (Throwable t) {
            Log.w(TAG, "releaseEglResources failed", t);
        } finally {
            latch.countDown();
        }
    });

    if (posted) {
        try {
            // 250 ms is plenty for glDelete*; we use a finite bound so a
            // wedged GL thread does not block onDestroy forever.
            latch.await(250, java.util.concurrent.TimeUnit.MILLISECONDS);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

## Field references

The names `yFbo`, `uFbo`, `vFbo`, `yTex`, `uTex`, `vTex`, `eglDisplay`,
`eglSurface`, `eglContext` match the existing `MainActivity.java` fields on
`main`. If the field names differ in your branch (e.g., a previous PR
renamed them), do a search-and-replace.

## Why a `CountDownLatch`?

`onDestroy` is on the main thread; the GL state is on `glThread`. We cannot
call `glDelete*` from the main thread because there is no current EGL
context. Posting to the GL thread and waiting (briefly, bounded) is the
safest cross-thread teardown for code that does not yet have a
`CaptureEngine` (step 04 introduces it and replaces this helper entirely with
`CaptureEngine.shutdown()`).

## Call sites for this helper

- `onDestroy` (sub-step 03.3).
- Any existing `cleanupCapture` path inside `MainActivity`. Replace inline
  GL deletes with a call to `releaseEglResources()`.

Find existing inline deletes:

```bash
rg -n 'glDeleteFramebuffers|glDeleteTextures|eglDestroy|eglTerminate' \
    app/src/main/java/org/fcast/android/sender/MainActivity.java
```

…and replace them with `releaseEglResources();`.
