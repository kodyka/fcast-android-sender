# 03.3 — Add `onDestroy`

Pair every register/start/acquire in `onCreate` with a matching
unregister/quit/release.

## New method

Add somewhere near the existing lifecycle methods (`onCreate`, `onResume`):

```java
@Override
protected void onDestroy() {
    // 1. Detach the CaptureResultBus listener (step 02 paired form).
    CaptureResultBus.setListener(null);

    // 2. Display listener — registered in onCreate.
    if (displayManager != null) {
        try {
            displayManager.unregisterDisplayListener(this);
        } catch (IllegalArgumentException ignored) {
            // Already unregistered. Safe to swallow.
        }
        displayManager = null;
    }

    // 3. MediaProjection callback — registered when capture starts.
    if (mediaProjection != null) {
        if (mediaProjectionCallback != null) {
            try {
                mediaProjection.unregisterCallback(mediaProjectionCallback);
            } catch (Exception e) {
                Log.w(TAG, "unregisterCallback failed", e);
            }
            mediaProjectionCallback = null;
        }
        try {
            mediaProjection.stop();
        } catch (Exception e) {
            Log.w(TAG, "mediaProjection.stop failed", e);
        }
        mediaProjection = null;
    }

    // 4. OpenGL thread — started in onCreate.
    if (glThread != null) {
        glThread.quitSafely();
        try {
            glThread.join(/* millis */ 1000L);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            Log.w(TAG, "glThread.join interrupted", e);
        }
        glThread = null;
        glHandler = null;
    }

    // 5. EGL / OpenGL resources — see 03.5.
    releaseEglResources();

    super.onDestroy();
}
```

## Ordering rationale

The order matters:

1. **CaptureResultBus listener first.** Prevents an in-flight
   `onCaptureResult` from re-arming GL state we are about to release.
2. **Display listener.** Stops any further `onDisplayChanged` callbacks that
   could touch `glHandler` after we null it.
3. **MediaProjection.** Tear down the projection before the GL thread quits
   so the projection's own callbacks land on a running looper.
4. **GL thread.** `quitSafely` drains pending messages — anything posted by
   steps 1–3 still completes.
5. **EGL resources.** Released on the same thread that owns them — see 03.5
   for the cross-thread handling.

## Fields that need null-resets

After this method:

- `glThread`, `glHandler`, `displayManager`, `mediaProjection`,
  `mediaProjectionCallback` are all null.
- Captures must check for these and short-circuit. Most existing checks
  already exist; if not, add a guard at each frame-arrival callback.
