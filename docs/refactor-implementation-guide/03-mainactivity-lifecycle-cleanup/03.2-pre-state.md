## EGL / OpenGL fields

```bash
rg -n 'EGL|GL_TEXTURE|glDeleteFramebuffers|glDeleteTextures' \
    app/src/main/java/org/fcast/android/sender/MainActivity.java | head
```

Verified field-style fragments (paraphrased — re-grep for the exact identifiers):

```java
private android.opengl.EGLDisplay eglDisplay;
private android.opengl.EGLSurface eglSurface;
private android.opengl.EGLContext eglContext;
private int[] yFbo  = new int[1];
private int[] uFbo  = new int[1];
private int[] vFbo  = new int[1];
private int[] yTex  = new int[1];
private int[] uTex  = new int[1];
private int[] vTex  = new int[1];
```

These are allocated lazily during the first frame. After `MediaProjection`
ends they remain in GL memory until the activity is destroyed. The new
`releaseEglResources()` helper (03.5) drops them deterministically.

## MediaProjection callback

The callback is currently registered inline in the code that starts capture
(after consent), but there is no explicit `mediaProjection.unregisterCallback(...)`
on stop. This step adds the unregister.
