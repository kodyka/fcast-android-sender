## Targeted leak hunt (LeakCanary not required)

Stress-test the lifecycle cleanup without LeakCanary:

```bash
for i in {1..10}; do
    adb shell am start -n org.fcast.android.sender/.MainActivity
    sleep 1
    adb shell input keyevent KEYCODE_BACK
    sleep 1
done

adb shell am force-stop org.fcast.android.sender
adb shell dumpsys meminfo org.fcast.android.sender | head -30
```

`Native Heap` and `EGL mtrack` should be stable across iterations. If they
climb monotonically, sub-step 03.5 (EGL release) is incomplete.

## When the test matrix is *not* enough

- A device whose vendor `MediaProjection` implementation calls back on a
  non-main thread may surface a `Looper.getMainLooper()` violation that
  doesn't reproduce on Pixels. If the issue manifests, add a `runOnUiThread`
  wrapper around the `unregisterCallback` call.
- A foldable that rapidly re-issues `onDisplayChanged` during the fold animation
  may surface a use-after-free in the GL thread. Reduce to a single
  `displayManager` reference and re-test.
