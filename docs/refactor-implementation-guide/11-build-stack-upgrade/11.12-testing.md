## Detailed: cold start measurement

Run before and after each PR:

```bash
for i in 1 2 3 4 5; do
    adb shell am force-stop org.fcast.android.sender
    sleep 1
    adb shell am start -W -n org.fcast.android.sender/.MainActivity \
        | grep TotalTime
done
```

Take the median of 5. Report in PR description:

> "Cold start: before 970 ms (median of 5), after 950 ms. Δ = -2%."

## Detailed: 60-second cast smoke

The minimum smoke is:

1. Launch the app.
2. Tap "Start cast" (or whatever the project's casting entry is).
3. Grant MediaProjection permission.
4. Cast for 60 seconds. Watch the receiver display real-time frames.
5. Tap "Stop cast".
6. Repeat steps 2–5 once more (caches warm; second run reveals
   shutdown leaks).

Pass criteria:

- No app crash.
- No native-lib crash.
- No memory leak visible in `adb shell dumpsys meminfo
  org.fcast.android.sender` (PSS should plateau, not grow).
- Receiver displays frames without obvious tearing or stutter.

## Stop-the-line conditions

Halt step 11 if:

- Symbol drift appears that can't be explained.
- APK size jumps > 10% on any single bump.
- Cold start regresses > 25%.
- Real-device smoke fails on any current-gen device (Pixel 6/7/8, Samsung
  S22+).
- GStreamer link errors appear that have no workaround.
