# 11.8 — PR 11.4: AGP 8.13 → 9.0

The major bump. Most invasive single PR in step 11.

## Diff

```diff
 # gradle/libs.versions.toml
 [versions]
- agp = "8.13.0"
+ agp = "9.0.1"
```

## What changes in AGP 9

| Change                                              | Action                                                                                                |
|-----------------------------------------------------|-------------------------------------------------------------------------------------------------------|
| Default NDK r25c → r28c                             | Either install r28c in CI (sub-PR 11.7), or pin `ndkVersion '25.2.9519653'` until 11.7 lands.          |
| `package` attribute removed from manifest          | Use `android.namespace` only. Audit `AndroidManifest.xml`.                                             |
| `buildFeatures.buildConfig` defaults to false       | If you reference `BuildConfig.X`, set `buildFeatures.buildConfig = true`.                              |
| Stricter manifest merger                             | Inline tools:replace="…" for any overrides.                                                            |
| `dependenciesInfo { … }` schema bump                  | Most projects don't notice.                                                                           |
| New `compileSdkPreview` for non-stable APIs         | Optional; defer.                                                                                       |
| ProGuard / R8 8.10+                                  | New optimisations; expect ~5–10% smaller release APK.                                                  |
| Removed legacy `transform` API                       | Custom build plugins may break; project has none.                                                      |
| Removed `compileOnly` for some configurations        | Replace with `compileOnlyApi` where transitive visibility is needed.                                   |
| Kotlin Symbol Processing (KSP) v2 default            | If KSP is used, may need version bump.                                                                  |

## Manifest changes

```bash
# Verify there's no `package` attribute
rg -n 'package="' app/src/main/AndroidManifest.xml
```

If present:

```diff
- <manifest xmlns:android="http://schemas.android.com/apk/res/android"
-     package="org.fcast.android.sender">
+ <manifest xmlns:android="http://schemas.android.com/apk/res/android">
```

Move the value to `app/build.gradle`:

```diff
 android {
+    namespace 'org.fcast.android.sender'
 }
```

## Override NDK pin (interim, until 11.7)

```diff
 android {
+    ndkVersion '25.2.9519653'
 }
```

This forces AGP 9 to use r25c regardless of the default. Remove in 11.7.

## BuildConfig

```bash
rg -n 'BuildConfig\.' app/src/main/
```

If anything references `BuildConfig.*`, enable the feature explicitly:

```diff
 android {
+    buildFeatures {
+        buildConfig = true
+    }
 }
```

## Smoke build

```bash
./gradlew --no-daemon :app:assembleDebug 2>&1 | tee /tmp/agp9.log
```

Expected initial failures (each is fixable):

- "`package` attribute is removed" — see above.
- "Cannot resolve `BuildConfig.X`" — see above.
- "`android.transform` API removed" — only if custom transforms exist
  (verify with `rg -n 'transform' app/build.gradle`).
- "Manifest merger failed" — likely a `tools:replace` needed somewhere.

For each error, apply the fix; commit; re-run.

## Symbol stability

After the build succeeds:

```bash
nm -D --defined-only target/aarch64-linux-android/debug/libfcastsender.so \
    | grep Java_org_fcast_android_sender | sort > /tmp/post-agp9.txt
diff /tmp/pre-agp9.txt /tmp/post-agp9.txt
```

Must be empty.

## Pre-merge

- [ ] Manifest audit: no `package` attribute.
- [ ] `BuildConfig` feature enabled if used.
- [ ] `ndkVersion '25.2.9519653'` pinned (until 11.7).
- [ ] Local: `:app:assembleDebug` green.
- [ ] Symbol diff empty (sub-step 09.10's guard).
- [ ] CI green.
- [ ] Slint headless tests green.
- [ ] Real device: launch + 60-second cast + lifecycle (rotate, background, foreground).

## Rollback

Single revert:

```bash
git revert <sha-of-11.4>
```

Returns to AGP 8.13. The 11.3 (compileSdk 35) and 11.5 (targetSdk) PRs are
independent; they don't need to revert with 11.4.

## Why "High risk"

AGP majors are the most invasive single-PR changes in the Android
ecosystem. The risk is mitigated by:

- Pinning NDK to r25c (no link-line change).
- Disabling configuration cache (Gradle 9 default; can re-enable as
  follow-up).
- Symbol-stability guard catches ABI drift.
- Real-device smoke catches lifecycle regressions.
