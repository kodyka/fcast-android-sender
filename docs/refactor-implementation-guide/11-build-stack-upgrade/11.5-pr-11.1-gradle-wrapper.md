# 11.5 — PR 11.1: Gradle wrapper 8.9 → 9.5

The smallest, safest upgrade. Gradle 9 is backward-compatible with AGP
8.x; this PR doesn't touch AGP.

## Steps

```bash
./gradlew wrapper --gradle-version 9.5.0 --distribution-type bin
```

Commits:

- `gradle/wrapper/gradle-wrapper.properties`
- `gradle/wrapper/gradle-wrapper.jar`

The `.jar` is binary; review by comparing the `distributionUrl` line in
the `.properties` file:

```diff
- distributionUrl=https\://services.gradle.org/distributions/gradle-8.9-bin.zip
+ distributionUrl=https\://services.gradle.org/distributions/gradle-9.5.0-bin.zip
```

## Verify

```bash
./gradlew --version
```

Output should report:

```

Build time: …
Kotlin: …
Groovy: …
Ant: …
Launcher JVM: 21 (…)
Daemon JVM: 21 (…)
```

## Smoke build

```bash
./gradlew --no-daemon :app:assembleDebug
```

Common Gradle 9 issues:

- Deprecation warnings for `compile` configuration (replace with
  `implementation` or `api`).
- Removed `useIR = true` (was deprecated since Kotlin 1.5; if any
  build.gradle still has it, delete).
- `kotlinOptions` block usage changes (the deprecation is benign for
  this PR; revisit in 11.6).

## Configuration cache

Gradle 9 enables configuration cache by default for new projects. The
existing project may not be compatible. To opt out for this PR:

```diff
 # gradle.properties
+org.gradle.configuration-cache=false
```

…then file a follow-up to migrate to config cache. Not in scope here.

## CI implications

`.github/actions/android-ci-setup/action.yml` doesn't pin a Gradle
version (it relies on the wrapper). The composite action's
`actions/setup-java@v4` step must accept Java 21:

```yaml
- name: Set up JDK 21
  uses: actions/setup-java@v4
  with:
    distribution: temurin
    java-version: '21'
```

If the existing composite uses Java 17, bump to 21 — Gradle 9 requires
Java 17+; the project's CI matrix is already on 21.

## Time impact

| Phase            | Before (Gradle 8.9)   | After (Gradle 9.5)    |
|------------------|------------------------|------------------------|
| Configuration    | ~6 s                   | ~4 s                   |
| `assembleDebug`  | ~110 s                 | ~100 s                 |
| Incremental rebuild | ~30 s              | ~25 s                  |

Modest gains; bigger wins come with configuration cache (follow-up).

## Configuration cache follow-up

`gradle.properties` disables config cache with:

```
org.gradle.configuration-cache=false
```

This is intentional for this PR. A follow-up task must be opened to
enable it after all plugins are verified compatible:

- [ ] Open task: "Enable Gradle configuration cache (follow-up to 11.1)"
- Target: after PR 11.2 (AGP 8.13) lands and CI is green on the new
  AGP version, as AGP 8.10+ has first-class config-cache support.

## Why this PR is "Low risk"

Gradle 9.x is designed to be drop-in for projects on AGP 8.x. The
only common breakage is custom build logic (Groovy DSL with deprecated
APIs), which this project does not appear to have extensively.

If something does break, revert is one commit (the wrapper) — no
runtime impact.
