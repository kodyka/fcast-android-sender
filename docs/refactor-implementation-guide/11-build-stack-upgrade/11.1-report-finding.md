## Why "High effort, Medium priority"

The version drift between the project and the current Android tooling is
real, but the cost of staying on AGP 8.7 / SDK 34 is limited to:

- New AGP features not yet available (e.g., better incremental
  compilation in 8.13+, baseline profile generation).
- Lint rules silently outdated.
- New deprecation warnings only visible after the upgrade.

None of these break shipping; they slow it down. Hence "Medium" priority
— the cost is paid in invisible developer-time, not user-visible bugs.

## Why "High effort"

Native + GStreamer + Slint combine to multiply the upgrade surface:

- Every Gradle/AGP/SDK/Kotlin/NDK bump is a separate experiment.
- GStreamer Android 1.28.0 is the linking authority; bumping NDK changes
  the ABI surface it expects.
- Slint's Android backend has its own AGP compatibility matrix (verify
  in `slint-android`'s changelog before each step).

The recipe in 11.5–11.11 sequences the bumps so each is observable in
isolation.

## Constraints

- **One bump per PR.** Combining Gradle + AGP + SDK in one PR makes
  bisection impossible if something breaks. This is the single most
  important rule of step 11.
- **CI must stay green at every intermediate state.** Don't ship 11.4
  if 11.3 is red — fix forward first.
- **Real-device smoke after each bump.** Emulator-only validation misses
  GStreamer link issues. Plug in a Pixel or Samsung and try a 60-second
  cast.
