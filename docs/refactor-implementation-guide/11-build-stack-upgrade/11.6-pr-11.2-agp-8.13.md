# 11.6 — PR 11.2: AGP 8.7 → 8.13

8.13 is the latest 8.x line. This PR stays in the 8.x line as the natural
intermediate before 9.x.

## Diff

```diff
 # gradle/libs.versions.toml
 [versions]
- agp = "8.7.0"
+ agp = "8.13.0"
```

## Why 8.13 and not 9 directly?

Skipping major versions multiplies bisection complexity. 8.7 → 8.13 → 9.0
gives three observation points; 8.7 → 9.0 gives one. If anything
regresses, knowing which delta caused it saves hours.

## What changes in 8.7 → 8.13

| Change                                               | Impact                                                     |
|------------------------------------------------------|------------------------------------------------------------|
| ProGuard / R8 8.4.x → 8.7.x                          | Improved string optimisation; possible APK size delta.      |
| `dependenciesInfo` block format                      | Minor manifest-merger tweaks; mostly invisible.              |
| Lint baseline upgrade                                 | New lint rules — expect 0–5 new warnings.                   |
| Baseline profile generation improvements             | If step 12 uses baseline profiles, performance ↑.           |
| `dexOptions { }` removal                              | If `dexOptions` exists in `app/build.gradle`, delete it (no-op). |

## Audit for `dexOptions`

```bash
rg -n 'dexOptions' app/build.gradle
```

If present, delete the entire block. It's been a deprecated no-op for
years.

## Audit for `compile` configuration

```bash
rg -n '\bcompile [\'"]' app/build.gradle
```

Replace any `compile 'X:Y:Z'` with `implementation 'X:Y:Z'` (or `api`
for transitive visibility).

## Smoke build

```bash
./gradlew --no-daemon :app:assembleDebug
```

Watch for:

- New `Notice` outputs from R8 (mostly informational).
- New lint warnings (resolve or baseline them).
- Missing `namespace` in submodules (AGP 8.13 stricter check).

## Lint baseline regeneration

If lint complains about new rules:

```bash
./gradlew :app:lintDebug
# Inspect app/build/reports/lint-results-debug.html
# If too noisy, regenerate the baseline:
./gradlew :app:updateLintBaseline
```

Don't suppress every new lint — review them. A new `UnusedResources`
lint, for instance, may surface real cleanup opportunities (step 12.x).

## Why "Medium risk"

AGP minor bumps occasionally introduce subtle behaviour changes (e.g.,
R8 optimisations that strip slightly different code paths). The risk is
mitigated by:

- Lint baseline absorbs new rules without breaking the build.
- ProGuard rules in `app/proguard-rules.pro` are the safety net for any
  R8 changes. The project's release variant likely doesn't enable
  minify (per sub-step 09.7); if it does, double-check the rules.
