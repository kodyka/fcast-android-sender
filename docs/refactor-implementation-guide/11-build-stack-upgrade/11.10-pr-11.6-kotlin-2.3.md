## What changes 2.0 → 2.3

| Change                                              | Impact                                                  |
|-----------------------------------------------------|---------------------------------------------------------|
| K2 compiler stable improvements                     | ~10% faster incremental compiles.                       |
| `data object` formalised                             | Already used in step 08; no source change needed.        |
| New `Result.toString()` semantics                    | Cosmetic; tests may need re-assertion.                   |
| Stricter null-safety in JVM interop                 | A handful of new compile errors; fix with `?` or `!!`.   |
| `kotlin.experimental.useK2 = true` is the default    | Remove the property if set.                              |
| `kotlin.code.style` defaults to `official`           | Re-run `ktlint` to confirm format.                       |

## Common compile errors in 2.0 → 2.3

- "Type 'X?' cannot be safely cast to 'X'" — add `?.let`.
- "Argument type mismatch: java.lang.String to kotlin.String" — add
  explicit `as String` or `requireNotNull(...)`.
- "Smart cast is impossible because the receiver is mutable" — assign
  to a local val first.

Fix one error, rebuild, fix next. The full Kotlin 2.3 migration is
usually 5–10 errors for a project this size.

## Coroutine API changes

`kotlinx-coroutines-test` 1.9 deprecated some APIs:

- `runBlockingTest` → use `runTest`.
- `TestCoroutineScope` → use `TestScope`.

If step 10's tests use the deprecated API, migrate:

```diff
- @OptIn(ExperimentalCoroutinesApi::class)
- class SenderControllerTest {
-     private val scope = TestCoroutineScope()
-     @Test fun foo() = scope.runBlockingTest { … }
- }
+ @OptIn(ExperimentalCoroutinesApi::class)
+ class SenderControllerTest {
+     @Test fun foo() = runTest { … }
+ }
```
