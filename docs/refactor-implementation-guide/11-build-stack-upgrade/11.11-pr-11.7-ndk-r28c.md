# 11.11 — PR 11.7: NDK r25c → r28c

The riskiest bump. NDK changes the native compiler, linker, libc++, and
ABI surface seen by GStreamer.

## Pre-validation (mandatory)

```bash
# On a feature branch, install r28c locally
mkdir -p /tmp/android-ndk && cd /tmp/android-ndk
curl -L -o ndk.zip \
    https://dl.google.com/android/repository/android-ndk-r28c-linux.zip
unzip -q ndk.zip
export ANDROID_NDK_ROOT=/tmp/android-ndk/android-ndk-r28c

# Try the GStreamer glue build
bash ci/build-gstreamer-android-glue.sh
```

If the GStreamer glue build succeeds, proceed. If it fails with linker
errors (`undefined reference to __cxa_*`, ABI mismatch), GStreamer 1.28.0
is incompatible with r28c. Stop this PR; wait for GStreamer 1.30.

## Smoke build

```bash
./gradlew --no-daemon :app:assembleDebug
```

Common failures:

- "Native library missing libc++_shared.so" — r28c uses a different
  layout. Update copy commands in `ci/build-rust-android-lib.sh` to
  point to the new path.
- "Link error: undefined reference to `__cxa_throw_bad_array_new_length`"
  — GStreamer 1.28 incompatibility. Pin back to r25c.
- "ld: error: cannot find -lc++_shared" — explicit library link line
  needs adjustment; check `RUSTFLAGS` in build script.

## Verify the .so loads on device

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
adb shell am start -n org.fcast.android.sender/.MainActivity
adb logcat -d *:E | grep -i 'unsatisfiedlink\|dlopen\|libfcastsender'
```

No errors → the .so loaded correctly. Common error:

```
java.lang.UnsatisfiedLinkError: dlopen failed: cannot locate symbol "__emutls_get_address"
```

…means an ABI mismatch. Revert to r25c.

## Symbol stability

```bash
nm -D --defined-only target/aarch64-linux-android/debug/libfcastsender.so \
    | grep Java_org_fcast_android_sender | sort > /tmp/post-r28c.txt
diff /tmp/pre-r28c.txt /tmp/post-r28c.txt
```

Must be empty. New `__cxa_*` runtime symbols are OK (they're not in the
`Java_…` namespace); a missing `Java_…` symbol is a regression.

## Performance impact

NDK r28c uses an updated clang. Expect:

- 0–5% smaller `.so` due to better LTO.
- 0–3% faster runtime (LLVM optimisations).
- Possibly different JIT warm-up profile (mostly invisible).

## Rollback

```bash
git revert <sha-of-11.7>
```

Returns to r25c. Re-bump the cache key to `r25c` if it was updated.

The pinned `ndkVersion '25.2.9519653'` in `app/build.gradle` (from
sub-PR 11.4) means revert restores the explicit pin too.

## Why "High risk"

NDK bumps are the riskiest single bump in this step because:

- They change the link line (potential ABI mismatch with GStreamer).
- They change the libc++ runtime (potential symbol mismatch on device).
- They change the optimisation profile (potential perf regression or
  improvement).

The mitigation is the pre-validation: building GStreamer glue against
r28c on a feature branch before merging the bump. If the pre-validation
fails, this PR doesn't ship until GStreamer catches up.
