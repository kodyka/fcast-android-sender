## High-impact risks in detail

### GStreamer Android 1.28.0 + NDK r28c

GStreamer prebuilts ship with a fixed NDK ABI target. 1.28.0 was built
against r25b/r25c. NDK r28c uses a newer libc++ runtime that, in some
configurations, fails to link with r25c-compiled object files.

**How to validate:**

1. Check out `main` on a fresh branch.
2. Install NDK r28c locally.
3. Set `ANDROID_NDK_ROOT` to the r28c path.
4. Run `bash ci/build-gstreamer-android-glue.sh`.
5. If linker errors appear (`undefined reference to __cxa_*`), GStreamer
   1.28.0 is incompatible — revert to r25c for now.

**Mitigation if incompatible:**

Pin `ndkVersion '25.2.9519653'` in `app/build.gradle` regardless of
AGP. AGP 9 respects the explicit `ndkVersion`; the default switch to
r28c only applies if unspecified.

### Android 16 orientation/resizability

API 36 changes large-screen behaviour:

- `android:resizeableActivity` defaults to `true` for new apps; existing
  manifests that explicitly set `false` continue to be honoured, but
  Play Store may flag this in the future.
- `screenOrientation="portrait"` on a foldable triggers the unfolded
  layout to stretch in unexpected ways.

**Mitigation:**

Audit the manifest after PR 11.5:

```bash
rg -n 'screenOrientation\|resizeableActivity\|configChanges' \
    app/src/main/AndroidManifest.xml
```

For each match, ensure the value reflects current intent:

```xml
<activity
    android:name=".MainActivity"
    android:resizeableActivity="true"
    android:screenOrientation="userPortrait"
    android:configChanges="orientation|screenSize|screenLayout|smallestScreenSize|keyboardHidden">
```

`userPortrait` is more flexible than `portrait` — allows reverse-portrait
when the user has rotated their device.

### AGP 9 manifest merger

AGP 9 enforces explicit `package` removal from `AndroidManifest.xml` (it
must live in `android.namespace`). Check:

```bash
rg -n 'package="org.fcast.android.sender"' app/src/main/AndroidManifest.xml
```

If found, remove the attribute and ensure `android.namespace = 'org.fcast.android.sender'`
in `app/build.gradle`. AGP 8.x tolerated both; AGP 9 errors.

### Slint Android backend

Check `crates/migration-runtime/Cargo.toml` and any other Cargo.toml
that depends on `slint-android` or `slint_android`. Each Slint major
release pins a Gradle/AGP range:

```bash
grep -A 2 'slint' crates/*/Cargo.toml
```

If the project pins Slint 1.10, verify Slint 1.10's release notes for
the AGP compatibility window. Bump Slint first if necessary.

### Symbol stability across toolchain bumps

NDK r25c → r28c changes the linker. Some symbols may be added (e.g.,
`__cxa_…` runtime support); the `Java_…` symbols themselves should be
unchanged.

Verify with sub-step 09.10's guard:

```bash
nm -D --defined-only target/aarch64-linux-android/debug/libfcastsender.so \
    | grep Java_org_fcast_android_sender | sort
```

Diff against the pre-upgrade list. Must be identical.

## Risk monitoring

After each sub-PR lands, watch for:

- **Crash rate.** Spike in `MediaProjectionManager` permission flow?
  Likely a manifest-attribute regression.
- **APK size.** Spike >5%? Likely R8 / ProGuard rule change.
- **Cold start time.** Spike >20%? Likely a baseline-profile regression.

Tie to step 13's rollout plan: pause at the next 25% increment if any
metric spikes.
