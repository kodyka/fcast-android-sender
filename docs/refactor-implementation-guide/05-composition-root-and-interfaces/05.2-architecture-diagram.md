Key invariants:

- `AppGraph` is constructed in `FcastApp.onCreate()` — the only `Application`
  subclass.
- `MainActivity` reads `AppGraph` via `(application as FcastApp).graph`.
- Nothing in `MainActivity` knows about `GstPopServiceBridge` or
  `MigrationRuntimeServiceBridge` after this step.
- The existing `GstPopServiceBridge` / `MigrationRuntimeServiceBridge`
  static classes stay — they are the JNI shim and the only piece that
  changes their owner.
- On the Rust side, `BACKEND` global is marked `#[deprecated]` and routed
  through `App`'s registry. All new code uses the registry.

## Slint-side coupling

Slint globals are owned by Rust. The Slint `start-gstpop-service` callback
implementation (in Rust) used to call:

```rust
crate::backend::current().start_gstpop()
```

After this step it calls:

```rust
crate::app().registry().get(MediaBackendKind::Gstpop).start()
```

…where `crate::app()` returns `&'static App` from a `OnceCell`.
