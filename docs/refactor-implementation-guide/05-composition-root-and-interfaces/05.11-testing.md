## Rust-side commands

```bash
cargo test -p fcastsender backend::registry::tests::install_and_get -- --exact
cargo test -p fcastsender backend::registry::tests::unset_kinds_remain_none -- --exact
cargo test -p fcastsender app::tests::production_has_migration_backend -- --exact
```

If any of these are not found, the new modules are not declared in
`src/backend/mod.rs` (`pub mod registry;`) or `src/lib.rs` (`mod app;`).

## Kotlin-side commands

```bash
./gradlew :app:compileDebugKotlin
./gradlew :app:lint
./gradlew :app:assembleDebug
```

## Manual smoke

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
adb shell am start -n org.fcast.android.sender/.MainActivity
adb logcat -d JniRuntimeBridge:V '*:S' | head
```

Logcat should not contain any "no gstpop backend installed" or "App not
initialised" errors during a normal session.

## Edge cases worth manually verifying

- **`backend::current()` called before `App::init`.** Should panic in tests
  (caught by the `expect("App not initialised")` in `app()`). Confirm by
  reverting the `app::init(...)` line in `android_main`, running tests, and
  observing the panic.
- **Multiple `App::init` calls.** Should log a `warn!` and keep the first.
  Test by adding a debug helper that calls `init` twice; expect one warning
  log and no panic.
