# 05.10 — Slint-bridge wiring

Slint globals are defined in `ui/bridge.slint`. The Rust side declares
callback handlers; this sub-step updates each handler to go through
`crate::app::app().registry()`.

## What does NOT change

The `ui/bridge.slint` file itself is untouched. The existing identifiers stay:

```slint
// ui/bridge.slint  (excerpt; unchanged)
export enum MediaBackendKind { migration, gstpop }
export enum AppState { Disconnected, Connecting, Connected, Casting, … }

export global Bridge {
    in-out property <MediaBackendKind> media-backend: MediaBackendKind.migration;

    callback start-gstpop-service();
    callback stop-gstpop-service();
    callback start-migration-runtime-service();
    callback stop-migration-runtime-service();

    callback start-casting(scale-width: int, scale-height: int, max-framerate: int);
    callback stop-casting();
    // …
}
```

These remain the contract. Only the Rust handler bodies change.

## What changes (Rust side)

Locate every `ui.global::<Bridge>().on_*` handler in the Rust source. On
`main`, most live near `android_main` in `src/lib.rs`. Step 07 moves them into
`src/application/mod.rs` and `src/jni_bridge/*`; until then they remain in
`lib.rs`.

```bash
rg -n 'ui\.global::<Bridge>\(\)\.on_' src/lib.rs | head -40
```

Each handler body that used to call `crate::backend::current()` now calls
`crate::app::app().registry()`:

```rust
// BEFORE (on main, abbreviated)
ui.global::<Bridge>().on_start_gstpop_service({
    move || {
        let _ = crate::backend::current().start();
    }
});

// AFTER
ui.global::<Bridge>().on_start_gstpop_service({
    move || {
        let registry = crate::app::app().registry();
        match registry.require(crate::backend::registry::BackendKind::Gstpop) {
            Ok(backend)   => { let _ = backend.start(); }
            Err(err_msg)  => tracing::error!("no gstpop backend installed: {err_msg}"),
        }
    }
});
```

Repeat for:

- `on_stop_gstpop_service`
- `on_start_migration_runtime_service`
- `on_stop_migration_runtime_service`
- `on_start_casting`        — uses `BackendKind::Migration` or `Gstpop` based on `bridge.media-backend`
- `on_stop_casting`         — same
- Any other `on_*` handler that historically called `backend::current()`.

## Bridging from the UI's `MediaBackendKind`

The Slint enum is exposed as a Rust enum with the same variant names
(case-sensitive):

```rust
fn slint_to_registry(k: MediaBackendKind) -> crate::backend::registry::BackendKind {
    match k {
        MediaBackendKind::Gstpop    => crate::backend::registry::BackendKind::Gstpop,
        MediaBackendKind::Migration => crate::backend::registry::BackendKind::Migration,
    }
}
```

Add this helper near the top of the file that wires the callbacks. Use it
inside `on_start_casting`:

```rust
ui.global::<Bridge>().on_start_casting({
    let ui_weak = ui.as_weak();
    move |w: i32, h: i32, fps: i32| {
        let Some(ui) = ui_weak.upgrade() else { return; };
        let kind = slint_to_registry(ui.global::<Bridge>().get_media_backend());
        let registry = crate::app::app().registry();
        match registry.require(kind) {
            Ok(backend) => {
                if let Err(e) = backend.start_capture(w, h, fps) {
                    tracing::error!("start_capture failed: {e}");
                }
            }
            Err(msg) => tracing::error!("no backend installed: {msg}"),
        }
    }
});
```

## Slint-side state mapping (for context)

The Slint UI's `Bridge.change-state(to:)` (defined at `bridge.slint:450`) is
called from the Rust handlers above. After this step, that call moves from
the post-`backend::current()` block to the post-`registry.require()` block —
it does not change semantically:

```rust
// inside on_start_casting:
ui.global::<Bridge>().invoke_change_state(AppState::Connecting);
```

This is purely a refactor: every Slint callback continues to dispatch the
same sequence of state changes, just via the new registry.

## Test impact

The headless Slint UI tests (`tests/ui_snapshots.rs`) still drive the Bridge
globals directly. Tests that *previously* relied on `backend::install(...)`
to swap in a fake must now call:

```rust
crate::app::init(/* test App with InMemoryRegistry pre-populated */);
```

…before the UI invokes anything. Coordinate this with the test rewrite in
step 10.
