# 05.9 — Deprecate the legacy globals

Until every callsite is migrated to `crate::app::app().registry()`, the old
`backend::current()` / `backend::install()` shims live on as thin wrappers
that delegate to the new registry. Mark them `#[deprecated]` so the compiler
warns at every legacy callsite.

## Diff

```diff
diff --git a/src/backend/mod.rs b/src/backend/mod.rs
@@
-static BACKEND: once_cell::sync::Lazy<RwLock<Arc<dyn MediaBackend>>> =
-    once_cell::sync::Lazy::new(|| RwLock::new(Arc::new(MigrationBackend::new())));
+#[deprecated(note = "Use crate::app::app().registry() — see refactor step 05.")]
+#[doc(hidden)]
+static BACKEND_LEGACY_GUARD: () = ();
@@
-pub fn current() -> Arc<dyn MediaBackend> { BACKEND.read().clone() }
-pub fn install(new_backend: Arc<dyn MediaBackend>) { *BACKEND.write() = new_backend; }
+/// Deprecated. Returns the *migration* backend from the new registry. Existing
+/// callers expected the migration backend by default — that contract is
+/// preserved during the soak window.
+#[deprecated(note = "Use crate::app::app().registry().require(BackendKind::…) — see refactor step 05.")]
+pub fn current() -> std::sync::Arc<dyn MediaBackend> {
+    crate::app::app()
+        .registry()
+        .require(crate::backend::registry::BackendKind::Migration)
+        .expect("legacy current() called before any backend installed")
+}
+
+#[deprecated(note = "Use crate::app::app().registry().install(BackendKind::…, backend) — see refactor step 05.")]
+pub fn install(new_backend: std::sync::Arc<dyn MediaBackend>) {
+    crate::app::app()
+        .registry()
+        .install(crate::backend::registry::BackendKind::Migration, new_backend);
+}
```

## What the compiler now prints

Every legacy callsite emits:

```
warning: use of deprecated function `current`: Use crate::app::app().registry().require(BackendKind::…) — see refactor step 05.
```

That is the inventory of work left for the cleanup PR. Track it via a CI
warning budget if you want to enforce removal:

```bash
RUSTFLAGS="-D warnings" cargo build -p fcastsender
```

…will block the build until every legacy callsite is migrated.

## Order of operations

1. Land 05.7 (registry) + 05.8 (App context) + 05.9 (deprecation) as one PR.
2. Open a follow-up PR that migrates every `current()` / `install(...)` call
   site to the new `app().registry()` form.
3. Once the follow-up lands and CI is green, open a third PR that removes
   `backend::current` / `backend::install` entirely.

Three PRs, but each one is small and focused. Do not combine them — the
deprecation-warning soak window is the safety net.

## Edge case: `current()` callers that expect a specific kind

A few callers might be implicitly relying on `current()` returning the gstpop
backend (e.g., after a `backend::install(GstpopBackend)` earlier). Audit:

```bash
rg -n 'backend::install\(' src/
```

Each `install(...)` callsite tells you which kind the legacy code "swapped
in". Replace those installs with explicit `registry().install(BackendKind::…)`
in the migration PR.
