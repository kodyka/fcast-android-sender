# 05.8 — Rust `App` context + `OnceCell` bootstrap

Create the single `App` struct that holds the registry (and, after step 06,
the secret store).

Create `src/app.rs`:

```rust
//! The process-wide App context. One instance per process; constructed by
//! the JNI bootstrap entry and accessed via [`app`].
//!
//! Replaces the process-global `BACKEND: Lazy<RwLock<…>>`; see
//! docs/refactor-implementation-guide/05-composition-root-and-interfaces/.

use std::sync::Arc;

use once_cell::sync::OnceCell;

use crate::backend::registry::{BackendKind, BackendRegistry, InMemoryRegistry};
use crate::backend::{MediaBackend, MigrationBackend};
// Step 06 adds:
// use crate::secret::SecretStore;

/// The composition root for the Rust crate. Constructed once during
/// android_main / JNI bootstrap.
pub struct App {
    registry: Box<dyn BackendRegistry>,
    // Step 06 adds:
    // secrets: Box<dyn SecretStore>,
}

impl App {
    /// Build the production App: in-memory registry with the migration backend
    /// pre-installed (matches the legacy default).
    pub fn production() -> Self {
        let registry = InMemoryRegistry::new();
        registry.install(BackendKind::Migration, Arc::new(MigrationBackend::new()));
        Self { registry: Box::new(registry) }
    }

    pub fn registry(&self) -> &dyn BackendRegistry { self.registry.as_ref() }
}

/// Process-global accessor — returns &'static App after bootstrap.
static APP: OnceCell<App> = OnceCell::new();

/// Bootstrap entry called from android_main (or test harness).
pub fn init(app: App) {
    if APP.set(app).is_err() {
        // Re-init is a bug — but don't crash the process; the existing
        // instance wins.
        tracing::warn!("App::init called twice; ignoring");
    }
}

/// Access the App. Panics if init() was not called first.
pub fn app() -> &'static App {
    APP.get().expect("App not initialised; call init() during JNI bootstrap")
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn production_has_migration_backend() {
        let a = App::production();
        assert!(a.registry().get(BackendKind::Migration).is_some());
        assert!(a.registry().get(BackendKind::Gstpop).is_none());
    }
}
```

## Bootstrap

`android_main` (currently inside `src/lib.rs:1421`; step 07 moves it to
`src/android_main.rs`) is the only place that calls `App::init`:

```diff
fn android_main(platform_app: PlatformApp) {
+    crate::app::init(crate::app::App::production());
     // …existing setup…
}
```

Important: this is the only callsite. Tests bypass `android_main` and must
call `app::init(App::production())` from a `#[test]` fixture or use the
unit-test API that takes an `&App` argument directly.

## Why `OnceCell` and not `Lazy`?

`OnceCell` is initialised once explicitly by `init(...)`. `Lazy` would auto-
initialise on first access, which means we couldn't reliably swap in a test
double. The explicit init path is also a forcing function: if `app()` panics
because `init` wasn't called, the test crash is loud.

## Why `&'static App` and not `Arc<App>`?

`OnceCell::get()` returns `&'static App`, which is cheaper than `Arc::clone`
on every call. The `App`'s fields are themselves `Box<dyn …>` trait objects
behind references, which is the standard pattern.

## How to read this from JNI sites

```rust

    let registry = crate::app::app().registry();
    let backend = match registry.require(BackendKind::Migration) {
```

## Slint-side consumers

Slint callbacks are wired in Rust via `MainWindow::on_…()` calls. Each of
those handlers now uses `crate::app::app().registry()` instead of
`crate::backend::current()`:

```rust
ui.global::<Bridge>().on_start_gstpop_service(move || {
    let registry = crate::app::app().registry();
    let backend = match registry.require(BackendKind::Gstpop) {
        Ok(b)  => b,
        Err(e) => {
            tracing::error!("no gstpop backend installed: {e}");
            return;
        }
    };
    backend.start();
});
```

Compare to the legacy form on `main`:

```rust
ui.global::<Bridge>().on_start_gstpop_service(move || {
    crate::backend::current().start();
});
```

Each Slint callback gains the same kind-explicit lookup. This is the most
disruptive part of the Rust diff — list every callback site in the PR
description.
