# 02.6 — Remove the `androidx.localbroadcastmanager` dependency

After 02.4 + 02.5, no production code references `LocalBroadcastManager`.
Remove the dependency.

## Verify zero references first

```bash
rg -n 'LocalBroadcastManager' app/src
rg -n 'localbroadcastmanager' app/src
```

Both must return zero matches. If either prints anything, stop — apply the
missing service- or activity-side change before removing the dependency.

## Diff

```diff
diff --git a/app/build.gradle b/app/build.gradle
@@
 dependencies {
-    implementation 'androidx.localbroadcastmanager:localbroadcastmanager:1.1.0'
     // …
 }
```

If the project declares the dependency through a version catalog (`libs.versions.toml`),
remove it from the catalog as well:

```diff
diff --git a/gradle/libs.versions.toml b/gradle/libs.versions.toml
@@
-localbroadcastmanager = "1.1.0"
@@
-androidx-localbroadcastmanager = { group = "androidx.localbroadcastmanager", name = "localbroadcastmanager", version.ref = "localbroadcastmanager" }
```

## Smoke check

```bash
./gradlew :app:dependencies | grep -i localbroadcast
```

Must print nothing. If it still prints, a transitive dependency is pulling it
in; run:

```bash
./gradlew :app:dependencies --configuration releaseRuntimeClasspath \
    | sed -n '/localbroadcast/,/^\s*$/p'
```

…to identify the transitive parent, then either:

- Exclude it (`exclude group: 'androidx.localbroadcastmanager'` on the
  offending dependency), or
- Skip the dependency removal and leave a comment in `app/build.gradle`
  explaining why.

`androidx.localbroadcastmanager:1.1.0` is the most recent published version
and is the only known transitive caller within AndroidX — there should not be
one.

## What survives

Nothing on-disk. The dependency is purely build-time.
