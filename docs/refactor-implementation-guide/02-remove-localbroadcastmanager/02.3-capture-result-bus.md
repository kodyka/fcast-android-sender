# 02.3 — `CaptureResultBus.java`

A single typed in-process channel. Java for now (step 08 may move it to
Kotlin once the Kotlin plugin is applied).

Create the file `app/src/main/java/org/fcast/android/sender/CaptureResultBus.java`:

```java
package org.fcast.android.sender;

import android.content.Intent;
import androidx.annotation.MainThread;
import androidx.annotation.NonNull;
import androidx.annotation.Nullable;

import java.util.concurrent.CopyOnWriteArrayList;

/**
 * Process-local, typed delivery channel for screen-capture consent results.
 *
 * Replaces the deprecated androidx.localbroadcastmanager.content.LocalBroadcastManager
 * channel that previously coupled ScreenCaptureService to MainActivity.
 *
 * Contract:
 *   - Producer (ScreenCaptureService) calls {@link #deliver(int, Intent)} after a
 *     successful MediaProjection consent dialog.
 *   - Consumer (MainActivity) registers via {@link #setListener(Listener)} during
 *     onCreate and removes via {@link #setListener(null)} during onDestroy.
 *   - One listener at a time. Replacing the listener silently drops the previous.
 *   - All delivery is on the main thread. Producer callers must already be on the
 *     main thread (services start there by default); the bus does not post.
 */
public final class CaptureResultBus {

    /** Result of the MediaProjection consent dialog. */
    public static final class Result {
        public final int resultCode;
        @NonNull public final Intent data;

        public Result(int resultCode, @NonNull Intent data) {
            this.resultCode = resultCode;
            this.data = data;
        }
    }

    public interface Listener {
        @MainThread
        void onCaptureResult(@NonNull Result result);
    }

    private static volatile @Nullable Listener listener;

    private CaptureResultBus() {}

    /** Replace the listener. Pass {@code null} to detach. */
    @MainThread
    public static void setListener(@Nullable Listener next) {
        listener = next;
    }

    /** Producer-side: deliver one result to whoever is registered, if anyone. */
    @MainThread
    public static void deliver(int resultCode, @NonNull Intent data) {
        Listener current = listener;
        if (current != null) {
            current.onCaptureResult(new Result(resultCode, data));
        }
    }

    /**
     * For tests only — assert that no listener is registered. Throws so that
     * leaks fail loudly in unit tests rather than silently dropping events.
     */
    public static void requireNoListener() {
        if (listener != null) {
            throw new IllegalStateException(
                "CaptureResultBus listener leaked: " + listener);
        }
    }
}
```

Notes:

- `volatile` is enough — there is exactly one listener slot, and reads and
  writes are 32-bit on every Android JVM.
- The intentional `CopyOnWriteArrayList` import is dropped because a single
  listener is sufficient. If multiple listeners are ever needed, swap in a
  `CopyOnWriteArrayList<Listener>` here in one place.
- `requireNoListener()` is the hook for step 10's unit tests.

## Why not `SharedFlow`?

`SharedFlow` is the right choice once Kotlin is applied (step 08). On the Java
shell this would require importing the Kotlin stdlib and `kotlinx-coroutines`
just for one event class. The typed callback is the minimum-machinery form
that satisfies the report's "no application-wide event bus" requirement.
