# 07.7 — PR 7.4: move `android_main`

`android_main` (lines 1421–2594 today, ~1170 LOC) is the entry point that
`slint_android` calls. Move its body to `src/android_main.rs`. Keep the
function signature byte-identical.

## Before

```rust
// src/lib.rs
#[unsafe(no_mangle)]
pub fn android_main(platform_app: slint_android::PlatformApp) {
    // …1170 LOC of body…
}
```

## After

```rust
// src/lib.rs
mod android_main;
pub use android_main::android_main;
```

```rust
// src/android_main.rs
//! Android entry point. Called by slint_android via JNI bootstrap.
//! Extracted from src/lib.rs as part of refactor step 07.4.

use slint_android::PlatformApp;

#[unsafe(no_mangle)]
pub fn android_main(platform_app: PlatformApp) {
    // …same 1170 LOC body…
}
```

## What the body does (recap, no changes here)

The body orchestrates:

1. **JavaVM capture.** `let vm = platform_app.activity_vm();`
2. **App composition.** `crate::app::init(crate::app::App::production(vm));`
   (step 05 + 06 wire-up).
3. **Config migration.** `crate::config::migration::migrate_config_file(...)`
   (step 06.8).
4. **Slint platform init.** `slint::platform::set_platform(...)`.
5. **UI bootstrap.** `let ui = MainWindow::new()?;`
6. **Application construction.** `let app = Application::new(ui.as_weak());`
7. **Wire callbacks.** `app.wire_slint_callbacks();`
8. **Run the event loop.** `ui.run()?;`

After this PR, all eight of these stay byte-identical — they just live in
`android_main.rs` instead of `lib.rs`.

## Re-export visibility

`#[unsafe(no_mangle)]` is on the function. Re-exporting with `pub use` does
**not** create a second symbol; the original is the one in the .so. Confirm:

```bash
nm -D --defined-only target/aarch64-linux-android/debug/libfcastsender.so \
    | grep android_main
```

Must show **one** `android_main` symbol.

## Why does `android_main` need `#[unsafe(no_mangle)]`?

`slint_android` invokes `android_main` by symbol name (it's the standard
name for an Android NativeActivity entry). Without `#[unsafe(no_mangle)]`,
the Rust mangling would obscure the name and the activity wouldn't start.

Moving the function to a different module does not change this — `no_mangle`
is what matters.

## What this PR does NOT do

- Does not change the body of `android_main` at all.
- Does not move sub-helpers used only inside `android_main` (they go with
  it; if there are any, they are now private to `src/android_main.rs`).
- Does not change the visibility (`pub` stays `pub`; `fn` stays the same
  return type).

## Net diff sketch

- `lib.rs`: ~1170 LOC removed.
- `android_main.rs`: ~1170 LOC added.
- Net: zero LOC delta; pure file move.
