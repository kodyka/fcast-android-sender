## Inventory

```bash

```

Catalogue every hit. Typical patterns to look for:

| Pattern                                              | Risk                                                                 |
|------------------------------------------------------|----------------------------------------------------------------------|
| `JObject::from_raw(ptr)` where `ptr` came from JNI   | If `ptr` is no longer alive, dangling reference.                       |
| `JavaVM::from_raw(jvm_ptr)`                          | Reconstruction of the VM by raw pointer; should come from `OnceCell`.  |
| `Box::from_raw` on a pointer passed across FFI       | Memory ownership unclear; document the contract.                       |
| `unsafe { transmute(...) }`                          | Re-interpretation of bytes; document the source layout.                 |

## Per-hit audit checklist

For each hit, decide one of:

1. **Replace with a safer borrow.**
   - `JavaVM::from_raw(ptr)` ŌćÆ `let vm = APP_VM.get().unwrap();`
     where `APP_VM: OnceCell<JavaVM>` is set once during bootstrap.
   - `JObject::from_raw(ptr)` from a value obtained from a `GlobalRef` ŌćÆ
     hold the `GlobalRef` and call `.as_obj()`.

2. **Document the invariant.**
   Add a `// SAFETY: ŌĆ”` comment that names the invariant. Example:

   ```rust
   // SAFETY: `ptr` is the raw pointer returned by JNI's
   //   `(*env)->NewGlobalRef(env, obj)` in MainActivity_attach_global. The
   //   global ref is alive for the lifetime of the FCastApp process.
   let obj = unsafe { JObject::from_raw(ptr) };
   ```

3. **Delete the `unsafe` block.**
   If the audit shows the call no longer needed (e.g., the helper above
   already wraps it), remove the inline `unsafe`.

## Recommended `OnceCell<JavaVM>`

Place in `src/jni_bridge/helpers.rs`:

```rust
use jni::JavaVM;
use once_cell::sync::OnceCell;

/// The Android VM handle. Set exactly once during android_main bootstrap.
static VM: OnceCell<JavaVM> = OnceCell::new();

pub fn init_vm(vm: JavaVM) {
    if VM.set(vm).is_err() {
        tracing::warn!("init_vm called twice; ignoring");
    }
}

pub fn vm() -> &'static JavaVM {
    VM.get().expect("JavaVM not initialised; call init_vm() from android_main")
}
```

`android_main` calls `init_vm(platform_app.activity_vm())` exactly once.
Every callsite that previously did `JavaVM::from_raw(ptr)` now calls
`crate::jni_bridge::helpers::vm()`.

## Recommended global-ref pattern

Where the code does:

```rust
let obj = unsafe { JObject::from_raw(some_ptr) };
// ŌĆ”use objŌĆ”
```

ŌĆ”and `some_ptr` came from a raw `jobject` captured during a previous JNI
call, hold a `GlobalRef` instead:

```rust
let global = env.new_global_ref(obj)?;
// ŌĆ”later, on a different thread / call ŌĆ”
let mut env = vm().attach_current_thread()?;
let obj = global.as_obj();
```

`GlobalRef` is `Send + Sync` and tracks lifetime explicitly via Drop.
