# 07.9 — PR 7.6: promote stable modules to crates

Only after PR 7.1–7.5 are merged and the layout has been stable for ≥ 1
release cycle.

## What to promote

| Module             | New crate                | Rationale                                                       |
|--------------------|--------------------------|-----------------------------------------------------------------|
| `command`          | `crates/runtime-core`    | Pure-Rust HTTP/probe logic; no Android deps; benchmarkable.      |
| `application`      | `crates/runtime-core`    | UI-state orchestration; no Android JNI; testable in JVM.         |
| `backend`          | `crates/runtime-core`    | `MediaBackend` trait + `BackendRegistry`.                        |
| `secret` (non-jni) | `crates/runtime-core`    | Trait + `InMemorySecretStore`; the `jni` module stays in `app/`. |
| `jni_bridge`       | `crates/android-jni`     | Depends on `runtime-core`; only crate to depend on `jni`.        |
| `platform`         | `crates/android-jni`     | Lives with `jni_bridge` because of `PlatformApp` coupling.        |
| `app`              | (stays in `fcastsender`) | The composition root sits where it gets used.                     |

`android_main.rs` stays in the top-level crate `fcastsender` because it is
the Android entry point.

Target length: **≤ 150 LOC**.

## Why keep the wafer in the top-level crate?

`#[unsafe(no_mangle)]` on a sub-crate function works, but having the wafer
in the top-level crate means:

- A single grep target (`rg "^pub extern" src/lib.rs`) lists every JNI
  symbol.
- The `.so` builds from one crate; `cargo build -p fcastsender` is the only
  command developers learn.

If a future workspace decision moves `android-jni` symbols into its own
.so, drop the wafers; until then, keep them.

## Why this is "last"

The crate promotion changes the *build graph*; it forces every CI runner
and dev environment to fetch and compile new crate edges. Land it only
after the contents have stabilised, so the workspace edit doesn't compound
with semantic changes.

## Net diff sketch

- New `Cargo.toml` files for `runtime-core` and `android-jni`.
- `git mv`-style relocation of `src/{command,application,backend,secret}` →
  `crates/runtime-core/src/`.
- `git mv` of `src/{jni_bridge,platform}` → `crates/android-jni/src/`.
- `lib.rs` shrinks to ≤ 150 LOC.
- Workspace `Cargo.toml` updated.
