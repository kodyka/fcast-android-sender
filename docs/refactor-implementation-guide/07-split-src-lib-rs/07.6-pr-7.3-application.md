# 07.6 — PR 7.3: carve up `Application`

The `Application` struct is the second-largest item in `lib.rs` (lines
667–1290, ~620 LOC). It owns:

- The Slint `MainWindow` weak handle.
- Backend bootstrap (mostly delegated to `app::App` after step 05).
- The status-items pipeline.
- The Slint callback wire-up.

Split it into focused modules.

## Side note: Slint MainWindow wiring

The `Application::wire_slint_callbacks` body contains every
`ui.global::<Bridge>().on_*` handler. After step 05, each handler goes
through `crate::app::app().registry()`. PR 7.3 should *not* re-edit those
handlers — it just moves them to the new file location.

If you discover a handler that still references `crate::backend::current()`
(i.e. step 05's migration PR didn't catch it), fix it in a separate PR
that depends on this one, **not** inline. Mixing the move with semantic
changes is what makes large refactors un-reviewable.
