## Symbol-diff CI gate

The mandatory ABI guard:

```bash
nm -D --defined-only target/aarch64-linux-android/release/libfcastsender.so \
    | grep Java_org_fcast_android_sender \
    | sort > /tmp/symbols.current.txt
diff /tmp/symbols.previous.txt /tmp/symbols.current.txt
```

The diff must be **empty** between successive PRs in step 07. The CI
runner stashes `symbols.previous.txt` from the last green build of `main`
and diffs the new build against it. If they differ, the PR has accidentally
renamed/dropped a symbol — block the merge.

Add this as a step in the CI pipeline (covered in step 09).

## Test before/after parity

```bash
cargo test -p fcastsender -- --list > /tmp/tests.before.txt
# … apply PR …
cargo test -p fcastsender -- --list > /tmp/tests.after.txt
diff /tmp/tests.before.txt /tmp/tests.after.txt
```

For PRs 7.1–7.5, the diff is just the *path prefix* of each test
(module path change). The set of test names should match — no test is
silently dropped.

For PR 7.6 (crate promotion), tests *move crates*:

```bash
cargo test --workspace -- --list > /tmp/tests.workspace.after.txt
```

Verify each previously-present test name still appears in the workspace
listing.

## Smoke-test script (`scripts/smoke.sh`)

After step 07 lands, add a smoke script that exercises the most fragile
paths post-refactor:

```bash
#!/usr/bin/env bash
set -euo pipefail
adb install -r app/build/outputs/apk/debug/app-debug.apk
adb shell am start -n org.fcast.android.sender/.MainActivity
sleep 5
adb shell am force-stop org.fcast.android.sender

# Symbol guard
nm -D --defined-only ./target/aarch64-linux-android/debug/libfcastsender.so \
    | grep -c Java_org_fcast_android_sender > /tmp/sym.txt
test "$(cat /tmp/sym.txt)" = "15"

echo "OK"
```

Step 09 wires this into CI.
