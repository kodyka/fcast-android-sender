# 07.5 — PR 7.2: split JNI symbols by callsite

Move each of the 15 `Java_…` bodies into the right submodule of `jni_bridge`.

## Pattern

For every symbol, replace its current `lib.rs` declaration with a re-export
wafer; move the body verbatim to the destination module.

### Example — `MainActivity_nativeGraphCommand`

Before (`src/lib.rs`):

```rust
#[unsafe(no_mangle)]
pub extern "C" fn Java_org_fcast_android_sender_MainActivity_nativeGraphCommand<'local>(
    mut env: jni::JNIEnv<'local>,
    _class: jni::objects::JClass<'local>,
    json: jni::objects::JString<'local>,
) -> jni::objects::JString<'local> {
    // …200 LOC of body…
}
```

After (`src/lib.rs`):

```rust
#[unsafe(no_mangle)]
pub extern "C" fn Java_org_fcast_android_sender_MainActivity_nativeGraphCommand<'local>(
    env: jni::JNIEnv<'local>,
    class: jni::objects::JClass<'local>,
    json: jni::objects::JString<'local>,
) -> jni::objects::JString<'local> {
    crate::jni_bridge::main_activity::native_graph_command(env, class, json)
}
```

After (`src/jni_bridge/main_activity.rs`):

```rust
use jni::JNIEnv;
use jni::objects::{JClass, JString};

pub fn native_graph_command<'local>(
    mut env: JNIEnv<'local>,
    _class: JClass<'local>,
    json: JString<'local>,
) -> JString<'local> {
    // …same 200 LOC body, with `crate::jni_bridge::helpers::*` calls…
}
```

Note: the *signature* of the moved function has no `#[unsafe(no_mangle)]`
and no `extern "C"`. It's a regular Rust function. The exported ABI lives
on the `lib.rs` wafer.

## Symbol diff check (mandatory)

Before this PR:

```bash
nm -D --defined-only target/aarch64-linux-android/debug/libfcastsender.so \
    | grep Java_org_fcast_android_sender | sort > /tmp/before.txt
```

After:

```bash
nm -D --defined-only target/aarch64-linux-android/debug/libfcastsender.so \
    | grep Java_org_fcast_android_sender | sort > /tmp/after.txt
diff /tmp/before.txt /tmp/after.txt
```

Diff must be **empty**. If not, you renamed something accidentally; revert
the affected hunk.

## Module-level docs

Each new file gets a top doc:

```rust
//! JNI bridge — symbols called from MainActivity.java.
//!
//! Function names mirror the Java method names (minus the `native` prefix
//! and converted to snake_case). lib.rs re-exports them with the
//! `Java_org_fcast_android_sender_MainActivity_*` symbol names.
//!
//! Extracted from src/lib.rs as part of refactor step 07.2.
```

## Net diff sketch

- `lib.rs`: ~1500 LOC removed; ~70 LOC of wafer re-exports added.
- `jni_bridge/main_activity.rs`: ~1100 LOC.
- `jni_bridge/discovery.rs`: ~50 LOC.
- `jni_bridge/gstpop_bridge.rs`: ~80 LOC.
- `jni_bridge/migration_bridge.rs`: ~80 LOC.

Round-trip total: ~3700 LOC of net file motion, but the diff is dominated
by deletions in `lib.rs` and additions in the destination files. Review by
diffing the function body before and after — they should be byte-identical
modulo import paths.

## What this PR does NOT do

- Does not change visibility or signatures.
- Does not touch `unsafe` blocks (that's PR 7.7).
- Does not promote modules to crates (PR 7.6).
- Does not move `android_main` (PR 7.4) or `Application` (PR 7.3).
