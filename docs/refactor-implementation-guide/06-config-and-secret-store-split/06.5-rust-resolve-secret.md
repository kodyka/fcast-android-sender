# 06.5 — `resolve_secret(alias)` helper

A thin convenience around `app().secrets().get(...)` that all runtime code
uses instead of touching the store directly.

Append to `src/secret/mod.rs`:

```rust
/// Convenience accessor — returns the secret as UTF-8 (typical for API keys).
pub fn resolve_secret_str(alias: &str) -> Result<String, SecretError> {
    let bytes = crate::app::app().secrets().get(alias)?;
    let s = bytes
        .as_str()
        .map_err(|e| SecretError::Backend(format!("not utf-8: {e}")))?;
    Ok(s.to_owned())
}

/// Convenience accessor for non-text secrets.
pub fn resolve_secret_bytes(alias: &str) -> Result<SecretBytes, SecretError> {
    crate::app::app().secrets().get(alias)
}
```

## Logging discipline

Never log the resolved secret. The `SecretBytes::Debug` impl already prevents
accidental string-format leaks, but the helper above returns a plain
`String` — so `tracing::debug!("api key: {api_key}")` would leak. Wrap the
return in a newtype if you suspect a future developer will log it:

```rust
pub struct ApiKey(SecretBytes);
impl ApiKey {
    pub fn as_str(&self) -> Result<&str, std::str::Utf8Error> {
        self.0.as_str()
    }
}
// And drop the `resolve_secret_str` helper.
```

This is over-engineering for the current scope. Add it only if a code review
finds a logging leak.
