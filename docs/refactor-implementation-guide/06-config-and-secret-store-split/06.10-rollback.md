# 06.10 — Rollback

## Single revert

```bash
git revert <sha-of-the-step-06-PR>
```

What this restores:

- `RuntimeConfig.gstpop_api_key: String` (plaintext field).
- `start_graph_runtime` reads `cfg.gstpop_api_key` directly.
- The Slint TextEdit handler writes plaintext into the JSON.
- The migration helper `src/config/migration.rs` goes away.
- `data/SecretStore.kt`, `data/AndroidSecretStore.kt`,
  `data/SecretStoreBridge.kt`, `src/secret/mod.rs`, `src/secret/jni.rs` go.

## What survives the revert

- **The encrypted SharedPreferences file remains on device.** That's fine —
  it's just an opaque blob. Future installs may either re-migrate (if
  step 06 is re-merged) or ignore the file (if the cleanup PR runs).
- **No data loss.** The plaintext is still in the rewritten JSON on disk?
  *No — the rewrite removed it.* This is the only loss vector:

  > After step 06 has run on a device, the plaintext `gstpop_api_key` field
  > is no longer in `migration-config.json`. If you revert step 06, the
  > legacy code path reads `cfg.gstpop_api_key`, which is now empty, and
  > returns `RuntimeError::MissingApiKey`. The user has to re-enter their
  > API key.

For an on-device rollback that *preserves* the plaintext, you would need to
ship a release with both code paths and a UI nudge to "re-enter your key".
This is annoying enough that we recommend not rolling back step 06 on
production traffic — fix forward instead.

## Partial rollback

If only the Kotlin side (`AndroidSecretStore` + `SecretStoreBridge`) causes
a problem (e.g. an unexpected ANR during `MasterKey.Builder().build()`),
keep the Rust trait + `InMemorySecretStore`. Set `AppGraph.secretStore` to
the in-memory implementation as a quick hot-fix:

```diff
 class AppGraph(private val appContext: Context) {
     val secretStore: SecretStore by lazy {
-        AndroidSecretStore(appContext)
+        InMemorySecretStore()
     }
 }
```

This degrades the security posture to pre-step-06, but in a controlled,
forwards-compatible way (the alias-keyed format stays the same; the bytes
just live in memory). Use only as a stopgap.

## Permanent forward-rollback

Once step 06 is solid:

1. Delete the `gstpop_api_key` field from `LegacyConfig` and the migration
   function. The struct accepts only `gstpop_api_key_alias`.
2. Bump the file version (add `"version": 2` to the JSON, gate the parser
   on it).
3. Drop `src/config/migration.rs` entirely.

This is the long-term cleanup; do it in a separate PR 2–3 releases after
step 06 lands.

## Quick on-device repair (worst case)

If a user complains "my API key is gone after the upgrade", the support
script is:

```text
1. Open the app.
2. Settings → API Key.
3. Re-enter the key.
4. Save.
```

Step 06's migration runs at most once. Once the plaintext is in the
Keystore, subsequent app starts read it transparently.
