# 06.2 — Pre-state on `main`

## Where the secret lives

```bash
rg -n 'gstpop_api_key|api_key' src/ app/ ui/
```

Verified hits (re-grep to refresh line numbers):

```
src/migration/runtime.rs:    let api_key = cfg.gstpop_api_key.clone();
src/migration/runtime.rs:    pub struct RuntimeConfig { … pub gstpop_api_key: String, … }
src/lib.rs:                  serde_json::from_str::<RuntimeConfig>(json) // includes gstpop_api_key
app/src/main/java/.../MainActivity.java: // forwards config JSON; doesn't read the key
```

The plaintext flows:

```
user-typed-key (Slint TextEdit)
    → Slint Bridge.set-gstpop-api-key
    → Rust callback
    → serde_json::to_string(&RuntimeConfig { gstpop_api_key, … })
    → MigrationRuntimeServiceBridge.start(context, configJson)
    → Service.onStartCommand(intent) reads "config"  // intent.getStringExtra
    → nativeStartMigrationRuntimeHost(configJson)
    → migration::runtime::start_graph_runtime(cfg)
```

At every step the key is in plaintext, including the intent extras. Some
launchers / OEMs surface intent extras in their debug logs.

## Where the JSON is persisted

```bash
rg -n 'fs::write|write_to_file|save_config' src/ | head
```

Verified path on `main`:

```
src/config/mod.rs:   let path = ctx.config_dir().join("migration-config.json");
src/config/mod.rs:   fs::write(&path, serde_json::to_string_pretty(&cfg)?);
```

So `/data/data/org.fcast.android.sender/files/migration-config.json` ends
up containing the key in cleartext.

## What needs to change

| Layer                            | Before                            | After                                        |
|----------------------------------|-----------------------------------|----------------------------------------------|
| `RuntimeConfig` struct (Rust)    | `gstpop_api_key: String`          | `gstpop_api_key_alias: String`                |
| Slint TextEdit handler           | writes `gstpop_api_key` in config  | writes alias + stashes key via SecretStore   |
| `start_graph_runtime`             | reads `cfg.gstpop_api_key`         | reads `app().secrets().get(&cfg.alias)`       |
| Service intent extra              | `"config"` (plaintext JSON)        | `"config"` (alias-only JSON, no secret)       |
| On-disk JSON file                  | contains the key                  | contains only the alias                       |
| Android Keystore                  | unused                             | holds `secret_alias → bytes`                  |

## Backup posture

`AndroidManifest.xml:7` declares `android:allowBackup="true"`. After this
step, the cleartext is gone from the JSON, but the Keystore-backed
`EncryptedSharedPreferences` would still be eligible for cloud backup unless
excluded. Add a `<full-backup-content>` XML rule (or set
`android:allowBackup="false"`) — see sub-step 06.6 for the rule.
