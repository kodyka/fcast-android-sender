## Why "Medium priority"

The Android side has **zero** verified automated tests today. Every
service contract regression has to be caught by manual smoke testing.
The cost of a regression on a foreground service is high (silent crash,
empty notification, dropped capture). Tests close that gap.

It's "Medium" — not "Highest" — because:

- Rust-side tests + Slint headless tests cover the hot paths today.
- Manual smoke can find bugs the tests would catch.
- The tests gate behind step 08 (Kotlin shell) for the controller tests,
  which means the controller-test PR can't land until 08.2 ships.

## Three test layers

1. **JVM unit tests** (`app/src/test/`). Run with `:app:testDebugUnitTest`.
   No emulator needed. Fast. Robolectric stubs Android APIs.
2. **Instrumentation tests** (`app/src/androidTest/`). Run on an emulator
   via `:app:connectedDebugAndroidTest`. Slow (~3 min per emulator boot).
   Tests real Android APIs.
3. **Slint headless tests** (`tests/ui_snapshots.rs`). Already exist; this
   step *reaffirms* the gate.

The unit tests run on every PR. The instrumented tests run on a nightly
schedule (or with an explicit label) to control runner cost.

## What this step does NOT do

- Does not test JNI bridges end-to-end on JVM. That requires an actual
  device or emulator with the native library loaded. Covered by 10.7.
- Does not test Slint UI directly from Android. Slint testing happens on
  the Rust side; Android tests stop at the Kotlin/Java boundary.
- Does not add coverage reporting. Codecov / Coveralls upload is a
  follow-up.
