# 10.9 — Gradle test dependencies

```diff
 android {
+    testOptions {
+        unitTests {
+            includeAndroidResources = true
+            returnDefaultValues = true
+        }
+    }
+    defaultConfig {
+        testInstrumentationRunner "androidx.test.runner.AndroidJUnitRunner"
+    }
 }

 dependencies {
+    // JVM unit tests
+    testImplementation 'junit:junit:4.13.2'
+    testImplementation 'org.mockito:mockito-core:5.12.0'
+    testImplementation 'org.robolectric:robolectric:4.13'
+    testImplementation 'org.jetbrains.kotlinx:kotlinx-coroutines-test:1.9.0'
+
+    // Instrumentation tests
+    androidTestImplementation 'androidx.test:core:1.6.1'
+    androidTestImplementation 'androidx.test:runner:1.6.2'
+    androidTestImplementation 'androidx.test:rules:1.6.1'
+    androidTestImplementation 'androidx.test.ext:junit:1.2.1'
+    androidTestImplementation 'androidx.test.ext:junit-ktx:1.2.1'
+    androidTestImplementation 'org.jetbrains.kotlinx:kotlinx-coroutines-test:1.9.0'
 }
```

## Why each dependency

| Dependency                                | Used by                                                | Note                                                              |
|-------------------------------------------|--------------------------------------------------------|-------------------------------------------------------------------|
| `junit:junit:4.13.2`                      | Both JVM and instrumented JUnit `@Test` annotations.    | JUnit 4 (not 5) — matches AGP / AndroidX conventions.              |
| `org.mockito:mockito-core:5.12.0`         | `spy`, `verify`, `never`, etc. in JVM tests.            | Avoid `mockito-android` for JVM tests.                             |
| `org.robolectric:robolectric:4.13`        | Service + manifest stubbing in JVM tests.              | Requires JVM 17.                                                  |
| `kotlinx-coroutines-test:1.9.0`           | `runTest`, `StandardTestDispatcher`, `setMain`.        | Same version as the production dep (sub-step 08.9).                |
| `androidx.test:core:1.6.1`                | `ApplicationProvider.getApplicationContext()`.          | Required for instrumentation tests.                                |
| `androidx.test:runner:1.6.2`              | `AndroidJUnitRunner`.                                   | Resolves `testInstrumentationRunner`.                              |
| `androidx.test:rules:1.6.1`               | `@Rule(ActivityScenarioRule)` (future).                  | Not used today; added for future tests.                            |
| `androidx.test.ext:junit:1.2.1`           | `@RunWith(AndroidJUnit4::class)`.                       | Standard runner.                                                    |
| `androidx.test.ext:junit-ktx:1.2.1`       | Kotlin-friendly JUnit ext.                              | Brings reified `<T>` helpers.                                       |

## Test variants

`testOptions.unitTests.includeAndroidResources = true` lets Robolectric
read the project's resources (R.string, R.drawable, R.layout). Without
this flag, `R.string.notification_title` is unresolved at test time.

`returnDefaultValues = true` makes unmocked methods on Android stub
classes return `null`/`0`/`false` instead of throwing
`RuntimeException("Stub!")`. This is the safety net while Robolectric
handles the rest.

## Robolectric JVM compatibility

Robolectric 4.13 requires JVM 17. Sub-step 08.9 sets `kotlinOptions.jvmTarget = '17'`;
verify the same for the test JVM:

```diff
 android {
     compileOptions {
         sourceCompatibility JavaVersion.VERSION_17
         targetCompatibility JavaVersion.VERSION_17
     }
 }
```

If you see "Robolectric requires Java 17" at test time, the gradle JDK
toolchain is older than the source/target. Set:

```diff
 android {
+    kotlinOptions { jvmTarget = "17" }
+    kotlin { jvmToolchain(17) }
 }
```

## Local invocation

```bash
./gradlew --no-daemon :app:testDebugUnitTest
./gradlew --no-daemon :app:connectedDebugAndroidTest  # requires running emulator
```

Tests output to `app/build/reports/tests/testDebugUnitTest/index.html` and
`app/build/reports/androidTests/connected/index.html` respectively.

## Common errors

- "Could not find Robolectric in any dependency configuration" — the
  block went under `androidTestImplementation` by mistake. Robolectric is
  JVM-side; use `testImplementation`.
- "android.app.Service: method <init>() not mocked" — missing
  `testOptions.unitTests.returnDefaultValues = true`.
- "java.lang.NoClassDefFoundError: kotlin/coroutines/test/TestScopeKt" —
  coroutines-test dep version mismatch. Bump to ≥ 1.9.0.
