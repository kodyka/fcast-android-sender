# 10.5 — `StatusParserTest.kt`

## Recommended refactor first

The cleanest implementation pulls `parseStatus(json: String): BackendStatus`
out of `JniRuntimeBridge` into a free object:

```kotlin
// app/src/main/java/org/fcast/android/sender/runtime/StatusParser.kt
package org.fcast.android.sender.runtime

import org.json.JSONException
import org.json.JSONObject

internal object StatusParser {
    /**
     * Parses a backend status JSON blob into [BackendStatus]. Unparseable
     * input is mapped to `state="error"` with the literal exception
     * message; the caller does not see exceptions.
     */
    fun parse(json: String): BackendStatus {
        return try {
            val obj = JSONObject(json)
            BackendStatus(
                state    = obj.optString("state", "unknown"),
                message  = obj.optString("message").takeIf { it.isNotEmpty() },
                detail   = obj.optJSONObject("detail")?.toString(),
            )
        } catch (e: JSONException) {
            BackendStatus(state = "error", message = e.message, detail = null)
        }
    }
}
```

…and have `JniRuntimeBridge` delegate to it.

If you don't want to extract, the test can reach the private method via
reflection (snippet below), but extraction is preferred.

## Test file (with extracted `StatusParser`)

Place at
`app/src/test/java/org/fcast/android/sender/runtime/StatusParserTest.kt`:

```kotlin
package org.fcast.android.sender.runtime

import org.junit.Assert.assertEquals
import org.junit.Assert.assertNull
import org.junit.Assert.assertNotNull
import org.junit.Test

class StatusParserTest {

    @Test
    fun running_yields_runningState() {
        val s = StatusParser.parse("""{"state":"running"}""")
        assertEquals("running", s.state)
        assertNull(s.message)
        assertNull(s.detail)
    }

    @Test
    fun error_withMessage_isPreserved() {
        val s = StatusParser.parse("""{"state":"error","message":"boom"}""")
        assertEquals("error", s.state)
        assertEquals("boom", s.message)
    }

    @Test
    fun detail_objectIsPreservedAsRawJson() {
        val raw = """{"state":"running","detail":{"x":1}}"""
        val s = StatusParser.parse(raw)
        assertNotNull(s.detail)
        assertEquals("{\"x\":1}", s.detail)
    }

    @Test
    fun unparseable_isMappedToError() {
        val s = StatusParser.parse("not json")
        assertEquals("error", s.state)
        assertNotNull(s.message) // contains a JSONException message
    }

    @Test
    fun emptyMessage_isCoercedToNull() {
        val s = StatusParser.parse("""{"state":"running","message":""}""")
        assertNull(s.message)
    }
}
```

## Test file (without extraction — reflection-based)

If the team prefers not to extract `StatusParser`, this fallback works:

```kotlin
package org.fcast.android.sender.runtime

import org.junit.Assert.assertEquals
import org.junit.Test

class StatusParserTest {

    private fun parse(json: String): BackendStatus {
        // Reflectively invoke the private parseStatus method.
        val bridge = JniRuntimeBridge.javaClass.getDeclaredConstructors().first().newInstance() as JniRuntimeBridge
        val m = JniRuntimeBridge::class.java.getDeclaredMethod("parseStatus", String::class.java)
        m.isAccessible = true
        return m.invoke(bridge, json) as BackendStatus
    }

    @Test fun running_yields_runningState() {
        assertEquals("running", parse("""{"state":"running"}""").state)
    }
}
```

The reflection variant is fragile (renaming the method breaks the test
silently). Extract `StatusParser` if you can.

## Why this test class is required

`BackendStatus` is consumed by `SenderController`. Every state transition
in `UiState` depends on `BackendStatus.state` being one of `running`,
`stopped`, `error`. Without this test, a JSON-key typo on the Rust side
silently maps to `unknown` and the UI sticks on "Starting" forever.

## Test characteristics

- **Speed:** ~50 ms total (no Robolectric needed — pure JVM).
- **Coverage:** 5 test methods, all paths covered.
- **Maintenance:** Extract once; tests stay readable.
