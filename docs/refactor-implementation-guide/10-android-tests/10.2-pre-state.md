## Ceiling of what JVM tests can cover

Things that JVM tests **can** cover (with Robolectric):

- Service `onStartCommand` contract (null intent, unknown action,
  granted/cancelled result).
- Notification builder output (channel ID, title, importance).
- JSON parsing logic (status, error format).
- Coroutine state transitions in `SenderController`.

Things that JVM tests **cannot** cover (need instrumentation):

- Actual `startForeground` succeeding (Robolectric only stubs).
- MediaProjection permission flow.
- JNI symbol resolution (requires the `.so` and a real JVM).
- Notification rendering in the system tray.

Sub-step 10.3 categorises every test into the right layer.
