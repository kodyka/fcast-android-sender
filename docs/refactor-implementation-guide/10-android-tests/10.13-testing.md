## Local test command cheat-sheet

```bash
# JVM only (fast):
./gradlew --no-daemon :app:testDebugUnitTest

# Specific test class:
./gradlew --no-daemon :app:testDebugUnitTest \
    --tests 'org.fcast.android.sender.shell.SenderControllerTest'

# Single method:
./gradlew --no-daemon :app:testDebugUnitTest \
    --tests 'org.fcast.android.sender.shell.SenderControllerTest.startBackend_running_yieldsConnected'

# Instrumented (slow, needs emulator):
./gradlew --no-daemon :app:connectedDebugAndroidTest

# Rust tests:
cargo test -p fcastsender

# Slint headless:
cargo test -p fcastsender --test ui_snapshots
```

## Pre-step-10 baseline

Before the test files land:

```bash
./gradlew --no-daemon :app:testDebugUnitTest 2>&1 | grep -E 'tests run\|tests completed'
```

Should report `0 tests`. If it reports more, an unrelated PR added tests
between this guide's writing and now — incorporate them into the count.

## Post-step-10 baseline

```bash
./gradlew --no-daemon :app:testDebugUnitTest 2>&1 | grep -E 'tests run\|tests completed'
```

Should report `≥ 15 tests` after every test file (10.4–10.6) lands.

## Stop-the-line conditions

Halt step 10 if:

- `SenderControllerTest` is flaky on `runTest` (intermittent).
  → Switch to `UnconfinedTestDispatcher` for the affected test only.
- Robolectric 4.13 incompatibility with JVM toolchain.
  → Pin to Robolectric 4.12 (one minor version back); document.
- Emulator boot fails on the CI runner.
  → Cache the AVD (sub-step 10.11) or fall back to a smaller emulator
    profile (`nexus_5`).
