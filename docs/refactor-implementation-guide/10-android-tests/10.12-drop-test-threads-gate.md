# 10.12 — Drop `--test-threads=1` (after step 07.5)

Step 07 sub-PR 7.5 eliminates the shared process-global atomics that
forced the single-threaded test gate. Once that ships, remove the gate
in `.github/workflows/gstpop-smoke.yml`.

## Diff

```diff
- # --test-threads=1 required: two ignored tests share process-global atomics
- run: cargo test backend::gstpop -- --include-ignored --test-threads=1
+ run: cargo test backend::gstpop -- --include-ignored
```

## Pre-condition

```bash
# Verify step 07.5 has landed by checking the location of the moved tests:
rg -n 'mod phase9_dispatch_tests' src/
# Empty result expected (tests moved to jni_bridge/helpers.rs).
```

If the tests are still in `src/lib.rs`, this sub-step is premature; ship
step 07.5 first.

## Why this matters

```bash
time cargo test backend::gstpop -- --include-ignored                # parallel
time cargo test backend::gstpop -- --include-ignored --test-threads=1  # serial
```

On a 4-core runner, the parallel version is roughly 2.5–3× faster. For
CI, that's a 1–2 minute saving per run.

## How to verify the parallel run works

After step 07.5 lands:

```bash
cargo test backend::gstpop -- --include-ignored --test-threads=8 2>&1 | tail -5
```

If tests pass with 8 threads, the global state is gone. If they fail
intermittently, a global mutable still exists — locate it with:

```bash
rg -n 'static .*: .*<.*Atomic.*\|static .*: .*<.*Mutex.*\|static .*: .*<.*RwLock.*' \
    src/ crates/
```

Audit each hit. Move single-process-state into the test fixture.

## Common patterns to refactor (if step 07.5 didn't catch them)

- **Lazy static + counter:** Move into a fixture struct constructed per-test.
- **OnceCell of a JavaVM:** Mock the `JavaVM` via a test trait.
- **Process-global log buffer:** Use a `LogRing` per-test instead of
  static.

## Rollback

If parallel runs surface a real concurrency bug that you can't fix
immediately:

```diff
- run: cargo test backend::gstpop -- --include-ignored
+ run: cargo test backend::gstpop -- --include-ignored --test-threads=1
```

…and file a follow-up issue. The serial run is a known-good fallback.
