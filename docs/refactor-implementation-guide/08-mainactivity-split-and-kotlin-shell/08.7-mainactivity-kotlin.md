# 08.7 — Full `MainActivity.kt`

Final form after PR 8.3:

```kotlin
package org.fcast.android.sender

import android.os.Bundle
import androidx.activity.ComponentActivity
import androidx.lifecycle.lifecycleScope
import kotlinx.coroutines.flow.collect
import kotlinx.coroutines.launch
import org.fcast.android.sender.capture.ScreenCaptureCoordinator
import org.fcast.android.sender.qr.QrScannerLauncher
import org.fcast.android.sender.runtime.BackendKind
import org.fcast.android.sender.shell.SenderController
import org.fcast.android.sender.shell.UiState

/**
 * Thin Slint host. Wires AppGraph to the SenderController; collects
 * UiState; everything else lives elsewhere.
 *
 * JNI symbols are declared as `external fun`; their names match what
 * the Rust side exports (see refactor step 07 — symbol stability is a
 * pre-merge gate).
 */
class MainActivity : ComponentActivity() {

    private lateinit var coordinator: ScreenCaptureCoordinator
    private lateinit var qr: QrScannerLauncher
    private lateinit var controller: SenderController

    private val captureCallbacks = object : ScreenCaptureCoordinator.CaptureCallbacks {
        override fun onPermissionRequested(intent: android.content.Intent) {
            startActivityForResult(intent, REQ_PROJECTION)
        }
        override fun onCaptureStarted(width: Int, height: Int) {
            controller.onCaptureStartedFromCoordinator(BackendKind.MIGRATION, width, height)
        }
        override fun onCaptureStopped() {
            controller.onCaptureStoppedFromCoordinator()
        }
        override fun onCaptureCancelled(reason: String) {
            controller.onCaptureStoppedFromCoordinator()
        }
    }

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        val graph = (application as FcastApp).graph
        coordinator = graph.newCaptureCoordinator(captureCallbacks)
        coordinator.attach()
        qr          = QrScannerLauncher(this, graph.runtime)
        controller  = SenderController(graph.runtime, coordinator, qr)

        lifecycleScope.launch {
            controller.uiState.collect { onUiStateChanged(it) }
        }
    }

    override fun onDestroy() {
        controller.shutdown()
        coordinator.shutdown()
        super.onDestroy()
    }

    @Deprecated("Use ActivityResultLauncher; see QrScannerLauncher.")
    @Suppress("DEPRECATION")
    override fun onActivityResult(requestCode: Int, resultCode: Int, data: android.content.Intent?) {
        super.onActivityResult(requestCode, resultCode, data)
        if (requestCode == REQ_PROJECTION) {
            CaptureResultBus.deliver(resultCode, data ?: android.content.Intent())
        }
    }

    private fun onUiStateChanged(state: UiState) {
        // Slint observes its own globals via the Rust shim (see step 08.8).
        // This hook is reserved for non-Slint affordances (analytics, etc.).
    }

    // ── JNI symbol shims ─────────────────────────────────────────────────
    // Names match Rust's `Java_org_fcast_android_sender_MainActivity_native*`.
    // Visibility is `public` (Kotlin default) so JNI can resolve them.
    external fun nativeBackPressed()
    external fun nativeCaptureStarted(width: Int, height: Int)
    external fun nativeCaptureStopped()
    external fun nativeCaptureCancelled(reason: String)
    external fun nativeProcessFrame(
        yBuffer: java.nio.ByteBuffer,
        uBuffer: java.nio.ByteBuffer,
        vBuffer: java.nio.ByteBuffer,
        width: Int,
        height: Int,
        timestampUs: Long,
    )
    external fun nativeQrScanResult(payload: String)
    external fun nativeGraphCommand(json: String): String

    companion object {
        init { System.loadLibrary("fcastsender") }

        const val REQ_PROJECTION = 1001
        const val ACTION_MEDIA_PROJECTION_STARTED = "org.fcast.android.sender.MEDIA_PROJECTION_STARTED"

        /** Used by JniRuntimeBridge to invoke the graph-command native shim. */
        @JvmStatic
        fun nativeGraphCommandStatic(json: String): String =
            // The instance-bound `nativeGraphCommand` is non-static; the
            // Rust side expects an *instance* call.  This helper bridges
            // step 05's JniRuntimeBridge.graphCommand to the real method.
            // For the soak window, route via the currently-foregrounded
            // activity (held by a process-global weak reference) — see 08.8.
            error("nativeGraphCommandStatic — provide a foregrounded MainActivity reference")
    }
}
```

## Why `ComponentActivity` and not `NativeActivity`?

`ComponentActivity` exposes `ActivityResultLauncher` and `lifecycleScope`.
`NativeActivity` doesn't.

If the project's Slint setup currently expects a `NativeActivity`,
`MainActivity` extends `androidx.activity.ComponentActivity` and the
slint_android shim either takes any `Activity` or you adapt by holding a
private `NativeActivityShim` field inside `MainActivity`. Verify with:

```bash
rg -n 'extends NativeActivity\|extends ComponentActivity\|class .*: NativeActivity\|class .*: ComponentActivity' \
   app/src/main/java/org/fcast/android/sender/MainActivity.java
```

If the existing class extends `NativeActivity`, keep that base class —
just convert the body to Kotlin. The `external fun` declarations work the
same way in either base class.

## JNI symbol resolution after conversion

```bash
nm -D --defined-only target/aarch64-linux-android/debug/libfcastsender.so \
    | grep MainActivity
```

…must list the same set of symbols as before the conversion. Kotlin
emits identical Java-class signatures for `external fun` declarations, so
this is automatic. Symbol-diff CI guard (step 07.3) catches any drift.

## `nativeGraphCommand` callsite migration

The companion-object `nativeGraphCommandStatic` is a wart: the Rust side
declares `Java_…_MainActivity_nativeGraphCommand` as an *instance* method.
`JniRuntimeBridge` calls it through a stored `MainActivity` reference (or
via `Looper`'s current main-thread context).

Cleaner long-term: make the JNI symbol `static` on the Java side too. Step
08 *does not* do that — it would break the symbol's Java-class signature.
Schedule it as a step-12 follow-up.

## Common compile errors during the conversion

- "Type mismatch: inferred type is Intent? but Intent was expected."
  → Kotlin null-checks default to non-null; use `?.let` or `requireNotNull`.
- "Unresolved reference: getSystemService."
  → use `getSystemService<X>()` extension from `androidx.core.content`.
- "Cannot inline bytecode built with JVM target 1.8 into bytecode that is
   being built with JVM target 17."
  → see `kotlinOptions { jvmTarget = '17' }` in `app/build.gradle`.
