# 04.4 — `CapturePermissionResult.kt`

A sealed result for the consent dialog so the coordinator handles each case
exhaustively.

Create `app/src/main/java/org/fcast/android/sender/capture/CapturePermissionResult.kt`:

```kotlin
package org.fcast.android.sender.capture

import android.content.Intent

/**
 * Outcome of the MediaProjection consent dialog (presented to the user the
 * first time they tap "start cast").
 *
 * Replaces the previous Bundle/Intent passing of `resultCode` + `data` that
 * leaked into half of MainActivity.
 */
sealed class CapturePermissionResult {
    /** The user tapped "Start now" — the engine can proceed. */
    data class Granted(val resultCode: Int, val data: Intent) : CapturePermissionResult()

    /** The user dismissed or cancelled the dialog. */
    data object Cancelled : CapturePermissionResult()

    /** Some other failure path (e.g. an `Intent.parseUri` error). */
    data class Failed(val reason: String, val cause: Throwable? = null) : CapturePermissionResult()
}
```

## Coordinator-side usage

```kotlin
private val captureListener = CaptureResultBus.Listener { result ->
    val outcome = if (result.resultCode == android.app.Activity.RESULT_OK) {
        CapturePermissionResult.Granted(result.resultCode, result.data)
    } else {
        CapturePermissionResult.Cancelled
    }
    handlePermissionResult(outcome)
}
```

…where `handlePermissionResult(outcome: CapturePermissionResult)` exhausts the
sealed hierarchy:

```kotlin
private fun handlePermissionResult(outcome: CapturePermissionResult) {
    when (outcome) {
        is CapturePermissionResult.Granted -> startEngine(outcome.resultCode, outcome.data)
        CapturePermissionResult.Cancelled -> callbacks.onCaptureCancelled("user")
        is CapturePermissionResult.Failed -> callbacks.onCaptureCancelled(outcome.reason)
    }
}
```

The `when` is exhaustive because the class is `sealed`; adding a new variant
later forces every caller to handle it (the compiler will reject the missing
branch).
