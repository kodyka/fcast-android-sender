# 04.3 — `CaptureConfig.kt`

A tiny, immutable value type for the parameters that come from the Slint UI
(scale width/height, max framerate). Living in `capture/` because the
coordinator and engine both consume it.

Create `app/src/main/java/org/fcast/android/sender/capture/CaptureConfig.kt`:

```kotlin
package org.fcast.android.sender.capture

/**
 * User-controlled capture parameters supplied by the Slint UI when the user
 * starts casting.
 *
 * @param scaleWidth Target output width in pixels. Zero / negative means
 *   "use the display's native width unchanged".
 * @param scaleHeight Target output height in pixels. Zero / negative means
 *   "use the display's native height unchanged".
 * @param maxFps Maximum number of frames per second to push through the
 *   engine. The engine drops frames that arrive faster than this. Must be > 0.
 *
 * Instances are intended to be created once per capture session and not
 * mutated. Use [copy] to derive variations.
 */
data class CaptureConfig(
    val scaleWidth: Int,
    val scaleHeight: Int,
    val maxFps: Int,
) {
    init {
        require(maxFps > 0) { "maxFps must be > 0, got $maxFps" }
    }

    val minIntervalNanos: Long
        get() = 1_000_000_000L / maxFps

    fun targetWidth(displayWidth: Int): Int =
        if (scaleWidth > 0) scaleWidth else displayWidth

    fun targetHeight(displayHeight: Int): Int =
        if (scaleHeight > 0) scaleHeight else displayHeight
}
```

## Why a `data class`?

- Immutable by convention; the `init` block enforces invariants.
- Comes with `equals`/`hashCode`/`copy` for free — useful in unit tests.
- Slint's `start-casting(scale-width: int, scale-height: int, max-framerate: int)`
  callback maps directly onto the constructor.

## Slint → Kotlin mapping

`ui/bridge.slint:328` declares:

```slint
callback start-casting(scale-width: int, scale-height: int, max-framerate: int);
```

In the Rust callback wiring (see step 05), construct `CaptureConfig` directly
from the three ints and forward it through JNI to the coordinator.

## Constants

If the existing `MainActivity` has implicit defaults (e.g. `maxFps = 30`),
hoist them as `CaptureConfig.Defaults` companion constants:

```kotlin
companion object {
    const val DEFAULT_MAX_FPS = 30
    val DEFAULT_NATIVE = CaptureConfig(scaleWidth = 0, scaleHeight = 0, maxFps = DEFAULT_MAX_FPS)
}
```

Only add these constants if they are read from more than one site.
