# 01.5 — Caller update in `MainActivity`

The service now requires `ACTION_RESULT` on its start intent. Update the one
call site in `MainActivity`.

## How to locate

```bash
rg -n 'new Intent\(this, ScreenCaptureService.class\)' \
    app/src/main/java/org/fcast/android/sender/MainActivity.java
```

This returns the single line that builds the start intent inside whichever
method handles `MediaProjectionManager`'s `onActivityResult` (search the file
for `mediaProjectionManager` if the line above doesn't find it).

## Diff

```diff
-            Intent svc = new Intent(this, ScreenCaptureService.class)
-                .putExtra("resultCode", resultCode)
-                .putExtra("data", data);
+            Intent svc = new Intent(this, ScreenCaptureService.class)
+                .setAction(ScreenCaptureService.ACTION_RESULT)
+                .putExtra("resultCode", resultCode)
+                .putExtra("data", data);
             startForegroundService(svc);
```

## Smoke check

```bash
./gradlew :app:assembleDebug
```

If the compile fails with "cannot find symbol ACTION_RESULT", sub-step 01.3
has not landed yet — apply that first.

## Note about other callers

There should be **no** other callers. Verify:

```bash
rg -n 'ScreenCaptureService' app/src/main
```

You should see only:

- `MainActivity.java` (one `new Intent(...)` builder).
- `AndroidManifest.xml` (the `<service>` declaration).
- `ScreenCaptureService.java` (the class itself).

If a third file appears, audit it the same way before merging this step.
