## 0. Goal of this step

Wire the three callbacks declared in STEP-3 (`apply-media-backend`,
`probe-media-backend`, `save-media-backend-settings`) to Rust
handlers that:

1. **Read** the settings page state from the `Bridge` global.
2. **Persist** the values to a JSON file under
   `<files-dir>/backend.json`.
3. **Construct** the right `MediaBackend` impl (Migration or GstPop).
4. **Install** it via `backend::install(...)`.
5. **Probe** the new backend and **push status** back to the Bridge
   so the settings page renders Ready / Error.

After this step the toggle is fully live: flipping it, pressing
Apply, and observing the status banner all work end-to-end.

## 1. Settings persistence

The app already persists user settings under the Android
context's files directory (the existing `BACKUP & RESET` callbacks
`export-settings` / `import-settings` cover the same area —
`ui/bridge.slint:210-214`). PHASE-12 piggybacks on that by writing a
small standalone file dedicated to backend selection.

Add `src/backend/persistence.rs`:

```rust
//! Persistence for the MediaBackend selector.
//!
//! Lives in <files-dir>/backend.json. Schema is a single object
//! suitable for human inspection / hand-editing.

use std::fs;
use std::path::PathBuf;

use anyhow::{Context, Result};
use serde::{Deserialize, Serialize};

use super::BackendKind;

#[derive(Clone, Debug, Default, Serialize, Deserialize)]
pub struct StoredBackendConfig {
    pub kind:            BackendKind,
    pub gstpop_url:      String,
    pub gstpop_api_key:  Option<String>,
    pub gstpop_pipeline_id: String,
}

impl StoredBackendConfig {
    pub fn defaults() -> Self {
        Self {
            kind:               BackendKind::Migration,
            gstpop_url:         "ws://127.0.0.1:9000".into(),
            gstpop_api_key:     None,
            gstpop_pipeline_id: "0".into(),
        }
    }

    pub fn load(files_dir: &PathBuf) -> Result<Self> {
        let path = files_dir.join("backend.json");
        if !path.exists() {
            return Ok(Self::defaults());
        }
        let bytes = fs::read(&path)
            .with_context(|| format!("read {path:?}"))?;
        serde_json::from_slice(&bytes)
            .with_context(|| format!("parse {path:?}"))
    }

    pub fn save(&self, files_dir: &PathBuf) -> Result<()> {
        let path = files_dir.join("backend.json");
        let json = serde_json::to_string_pretty(self)
            .context("serialize backend.json")?;
        fs::write(&path, json)
            .with_context(|| format!("write {path:?}"))?;
        Ok(())
    }
}
```

> **Why a separate file, not a key in the existing prefs?** The
> backend selector is small (4 fields), changes infrequently, and is
> intentionally scoped narrowly so a corrupt prefs blob can't
> cascade into "the app boots with no backend". Reloading at startup
> falls back to `defaults()` on any error.

### 1.1 Resolving `files-dir` on Android

`src/lib.rs` already resolves the Android files directory from the
JNI `Context` at boot (see how cast-history persistence does it).
For PHASE-12, plumb that path into `BackendLifecycle::new` (see §3)
rather than calling `ndk_context::android_context()` ad-hoc — the
former is unit-testable, the latter requires a live Android runtime.

## 2. `BackendLifecycle` — the handler glue

Add `src/backend/lifecycle.rs`:

```rust
//! Owns the wiring between Slint callbacks (PHASE-12 STEP-3) and
//! Rust backend impls (PHASE-12 STEP-5 / STEP-7).
//!
//! Pattern matches PHASE-9's start-migration-server / run-migration-test
//! handlers — each callback spawns a tokio task, the task does the
//! work, then schedules a Bridge-property writeback on the main
//! event loop via `slint::Weak::upgrade_in_event_loop`.

use std::path::PathBuf;
use std::sync::Arc;

use anyhow::Result;
use slint::{ComponentHandle, Weak};

use super::persistence::StoredBackendConfig;
use super::{
    install, BackendKind, BackendStatus, GstPopBackend, MediaBackend,
    MigrationBackend,
};
use crate::MainWindow; // generated by include_modules!

pub struct BackendLifecycle {
    files_dir: PathBuf,
}

impl BackendLifecycle {
    pub fn new(files_dir: PathBuf) -> Self {
        // Restore persisted selection at boot.
        if let Ok(stored) = StoredBackendConfig::load(&files_dir) {
            install_from_stored(&stored);
        }
        Self { files_dir }
    }

    /// Install handlers for the three PHASE-12 callbacks.
    pub fn register(self: Arc<Self>, ui: &MainWindow) {
        let bridge = ui.global::<crate::Bridge>();
        let weak   = ui.as_weak();

        // ── apply-media-backend ───────────────────────────────────────
        let lc = Arc::clone(&self);
        let weak_apply = weak.clone();
        bridge.on_apply_media_backend(move || {
            let lc       = Arc::clone(&lc);
            let weak_for_task = weak_apply.clone();
            tokio::spawn(async move {
                let config = read_config_from_bridge(&weak_for_task);
                if let Err(err) = lc.apply(config, weak_for_task.clone()).await {
                    push_error(&weak_for_task, &err.to_string());
                }
            });
        });

        // ── probe-media-backend ───────────────────────────────────────
        let weak_probe = weak.clone();
        bridge.on_probe_media_backend(move || {
            let weak_for_task = weak_probe.clone();
            tokio::spawn(async move {
                push_state(&weak_for_task, BackendState::Probing);
                let backend = super::current();
                match backend.probe().await {
                    Ok(status) => push_status(&weak_for_task, status),
                    Err(err)   => push_error(&weak_for_task, &err.to_string()),
                }
            });
        });

        // ── save-media-backend-settings ──────────────────────────────
        let lc_save = Arc::clone(&self);
        let weak_save = weak.clone();
        bridge.on_save_media_backend_settings(move || {
            let lc       = Arc::clone(&lc_save);
            let weak_for_task = weak_save.clone();
            tokio::spawn(async move {
                let config = read_config_from_bridge(&weak_for_task);
                if let Err(err) = config.save(&lc.files_dir) {
                    push_error(&weak_for_task, &format!("save failed: {err}"));
                }
            });
        });
    }

    async fn apply(
        &self,
        config: StoredBackendConfig,
        weak:   Weak<MainWindow>,
    ) -> Result<()> {
        // 1. Persist first so a crash mid-apply doesn't drop the
        //    user's choice on the floor.
        config.save(&self.files_dir)?;

        // 2. Install the new backend.
        install_from_stored(&config);

        // 3. Probe and write status back.
        push_state(&weak, BackendState::Probing);
        let backend = super::current();
        match backend.probe().await {
            Ok(status) => push_status(&weak, status),
            Err(err)   => push_error(&weak, &err.to_string()),
        }
        Ok(())
    }
}

fn install_from_stored(stored: &StoredBackendConfig) {
    let backend: Arc<dyn MediaBackend> = match stored.kind {
        BackendKind::Migration => Arc::new(MigrationBackend::new()),
        BackendKind::GstPop    => Arc::new(GstPopBackend::new(
            stored.gstpop_url.clone(),
            stored.gstpop_api_key.clone()
                .filter(|s| !s.is_empty()),
            stored.gstpop_pipeline_id.clone(),
        )),
    };
    install(backend);
}

// ── Bridge ↔ StoredBackendConfig helpers ─────────────────────────────

fn read_config_from_bridge(weak: &Weak<MainWindow>) -> StoredBackendConfig {
    let ui = match weak.upgrade() {
        Some(ui) => ui,
        None     => return StoredBackendConfig::defaults(),
    };
    let bridge = ui.global::<crate::Bridge>();
    StoredBackendConfig {
        kind:               super::from_slint(bridge.get_media_backend()),
        gstpop_url:         bridge.get_gstpop_url().to_string(),
        gstpop_api_key:     {
            let s = bridge.get_gstpop_api_key().to_string();
            (!s.is_empty()).then_some(s)
        },
        gstpop_pipeline_id: bridge.get_gstpop_pipeline_id().to_string(),
    }
}

enum BackendState {
    Disconnected,
    Probing,
    Ready,
    Error,
}

fn push_state(weak: &Weak<MainWindow>, state: BackendState) {
    let _ = weak.upgrade_in_event_loop(move |ui| {
        let bridge = ui.global::<crate::Bridge>();
        bridge.set_media_backend_state(match state {
            BackendState::Disconnected => crate::MediaBackendState::Disconnected,
            BackendState::Probing      => crate::MediaBackendState::Probing,
            BackendState::Ready        => crate::MediaBackendState::Ready,
            BackendState::Error        => crate::MediaBackendState::Error,
        });
    });
}

fn push_status(weak: &Weak<MainWindow>, status: BackendStatus) {
    let _ = weak.upgrade_in_event_loop(move |ui| {
        let bridge = ui.global::<crate::Bridge>();
        bridge.set_media_backend_state(crate::MediaBackendState::Ready);
        bridge.set_media_backend_status_text(status.status_text.into());
        bridge.set_media_backend_error_text("".into());
    });
}

fn push_error(weak: &Weak<MainWindow>, message: &str) {
    let message = message.to_owned();
    let _ = weak.upgrade_in_event_loop(move |ui| {
        let bridge = ui.global::<crate::Bridge>();
        bridge.set_media_backend_state(crate::MediaBackendState::Error);
        bridge.set_media_backend_error_text(message.into());
    });
}
```

### 2.1 Why the per-callback `tokio::spawn`

The callback bodies execute on Slint's main event-loop thread.
Doing any blocking work (let alone an async WS connect) there would
freeze the UI. Spawning a tokio task offloads the work and writes
back via `upgrade_in_event_loop`, the same idiom every PHASE-8 and
PHASE-9 handler uses (see `src/lib.rs`'s `on_recast`,
`on_run_quick_action`, `on_start_migration_server`).

### 2.2 Why `weak.upgrade_in_event_loop` (not `weak.upgrade`)

`upgrade` returns `Option<Strong<MainWindow>>` and must be called
*from* the event-loop thread. `upgrade_in_event_loop` re-enters the
event loop from a background thread — it's the correct primitive for
"I'm in tokio land, but I want to mutate a Slint property".

### 2.3 Why `apply` saves *before* installing

If the OS kills the process between `install_from_stored(&config)`
and `config.save(...)`, the next boot would restore the old config
and silently revert the user's choice. Saving first means the
choice survives any crash from that point forward.

### 2.4 Why we don't `await` Slint property writes

`upgrade_in_event_loop` returns a `Result<(), EventLoopError>` that
fires immediately. The closure is scheduled, but the *task we're in*
doesn't wait for it. That's fine — the next probe runs on the next
tokio tick, by which point the UI has already drawn the new state.

let files_dir = crate::resolve_android_files_dir()?;
